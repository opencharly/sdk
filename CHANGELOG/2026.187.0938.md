# Remove migration.cue from the SDK schema (moved to candy/plugin-migrate)

The `#Migration` validation schema is consumed ONLY by `candy/plugin-migrate` and
by no charly core code, so per the kernel/plugin boundary law it belongs in the
plugin, not the SDK contract.

- Delete `schema/migration.cue` — the `#Migration`/`#Scope`/`#MigrationOp`/… defs
  move to `candy/plugin-migrate/schema/migration.cue` (superproject).
- `internal/schemagen/main.go`: drop the now-dead `migration.cue` clause from
  `excludeParamGen` (the file no longer lives in the SDK FS).
- `#CanonCalVer` STAYS here in `version.cue` (the SDK's single source); the plugin
  concatenates its own `migration.cue` with the SDK `version.cue` at compile.

No authored wire-key change (the defs are `@go(-)`), so NO `#SchemaVersion` bump
and NO migration-table entry. `task cue:gen` is a no-op for every generated file;
`TestGenReproducible` stays green.
