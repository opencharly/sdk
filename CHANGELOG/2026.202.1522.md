## What this leg is

The sdk half of the FLOOR-SLIM Wave-1 super batch (core-minimization program):
four implementation units consolidated on `feat/floorslim-wave1`, receiving the
relocated kind-blind mechanisms, pure resolvers, and wire types that leave
`charly/` core in the paired superproject leg.

### New leaf package: `sdk/proclifecycle`

`sdk/vmshared/cleanup.go` (~240 LOC of 100%-stdlib process-lifecycle plumbing —
`RegisterShutdownHook`/`RegisterTempCleanup`/`UnregisterTempCleanup`/
`InstallSignalHandler`/`SweepStaleTemps`) carved into its own stdlib-only leaf
package, mirroring the `sdk/schemaconcat` precedent. It never had VM-specific
logic, and its real consumers span plugin-enc/oci/secrets/vm,
`deploykit/localpkg.go`, `vmshared/cloud_init_iso.go`, and charly's `main.go` —
all repointed to `proclifecycle.*` directly. `vmshared`'s dead
`AllDigits = allDigits` re-export deleted (zero consumers).

### The Config restructure (Unit 5)

`charly`'s `Config` type is now **`spec.Config`** (`sdk/spec/config.go` +
`spec/config_boxes.go`), with its method set split by type-graph direction:

- Purely spec-typed methods ride the type into `spec`: `BoxConfig`/`HasBox`/
  `SetBox`/`AllBoxNames`/`EachBox`, `ResolveBoxRef`/`FindBoxByLeaf`,
  `WalkBaseChain`/`WalkBaseChainDistro`/`WalkBaseChainBuild`/`BaseChainNode`,
  `ShouldIncludeDisabled`.
- Every **buildkit-typed** resolver became a FREE FUNCTION in
  `sdk/buildkit/config_resolve.go` taking `*spec.Config` (`ResolveBox`,
  `ResolveAllBox`, `resolveBase`/`Platforms`/`Tag`/`Distro`/`Build`,
  `ResolveEffectiveBuilder`/`EffectiveBuilderForBox`/`DistroBuilderMap`,
  `ResolveNamespacedBases`/`PullNamespacedBox`) — `spec` imports NEITHER
  `buildkit` nor `deploykit` (verified invariant), and `buildkit` carries ZERO
  `LoadUnified` coupling: callers pre-populate `DistroCfg`/`BuilderCfg`.
- `boxCandyChain`/`boxDirectCandies` → `deploykit.BoxCandyChain`/
  `BoxDirectCandies` (`deploykit/config_candy_chain.go`) — their real
  dependency is `deploykit.ResolveCandyOrder`, and deploykit is the top layer.

### The CheckResult wire envelope (Unit 4, ruling R-FS2/R-FS3)

- **New `sdk/schema/checkresult.cue` → `spec.CheckResult`** (CUE-sourced):
  every field EXCEPT the P12-spike-proven inexpressible exception,
  `DeadlineExceeded bool json:"-"`.
- **`spec/status_result.go`**: `Status` int enum + `String()` — CUE owns the
  wire value set, Go owns the formatting (gengotypes cannot express
  iota+Stringer).
- **`kit.CheckResult`** is now `struct { spec.CheckResult; DeadlineExceeded
  bool `json:"-"` }` — an embedding wrapper keeping the documented exception
  exactly spike-width. `kit.Status`/`StatusPass`/`Fail`/`Skip` stay as aliases
  onto `spec.*` so ~15 external plugin candies compile unchanged.
- **`kit.VerbResolver`** returns `spec.CheckResult` — what lets charly's
  registry-coupled floor files reference spec directly with zero new kit
  imports.

**Breaking wire-format change (`!`)** for `charly check box/live/run
--format json`/TAP consumers — deliberate snake_case consistency fix:

| Old key | New key |
|---|---|
| `Op` | `op` |
| `Verb` | `verb` |
| `Status` | `status` |
| `Message` | `message` |
| `Elapsed` | `elapsed` |

`attempts`/`total_elapsed`/`captured_value` unchanged; `DeadlineExceeded`
never crossed the wire and still does not. Golden proof:
`sdk/kit/checkresult_wire_test.go` (`TestCheckResult_WireParity_Populated`/
`_Zero`) — the diff against the former shape is EXACTLY this key rename (same
field set, values, omission behavior).
