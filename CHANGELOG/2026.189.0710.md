# 2026.189.0710 — keystone: InstallContext + BuildStageContext CUE-sourced → sdk/spec (P2/S1)

Moves the two build-render context keystones — `InstallContext` (format
install/prepare/cleanup templates) and `BuildStageContext` (builder multi-stage
templates) — from hand-written charly structs into the CUE single source
`sdk/schema/buildctx.cue`, GENERATED into `sdk/spec/cue_types_gen.go` by
`task cue:gen` (SDD — not a hand-written wire file). The `Repos` field's
list-of-open-struct shape generates faithfully as `[]map[string]any`; `[]#CacheMount`
→ `[]CacheMount`; `uid`/`gid` forced to Go `int` via `@go(,type=int)`; dual
yaml+json tags are automatic (config- AND plugin-wire-compatible). Aliased in
`vmshared/spec_aliases.go` (`BuildStageContext`/`InstallContext`), so charly's
Containerfile generator + template funcs reference them unchanged. First unit of
the P2 keystone migration (spike S1); the reproducibility gate (`TestGenReproducible`)
now covers them — a re-gen is a no-op, drift is an R1 incident. Establishes the
CUE-source-the-wire-types pattern (the `*_wire.go` family's classified future
schema-sourcing); genuine CUE-disjunction types remain hand-written (`@go(-)`).
