# 2026.189.0059 — LibvirtGraphicsListeners JSON tri-shape (opaque-path regression)

Cutover L de-typed the vm substrate body to opaque JSON decoded into `spec.Vm`,
but `spec.LibvirtGraphicsListeners` carried only an UnmarshalYAML + a charly-side
YAML expander — no `UnmarshalJSON`. So the documented `listen: 127.0.0.1` scalar
shorthand (and the object/list forms) failed on the JSON path with
"cannot unmarshal string into … LibvirtGraphicsListeners" (the
`check-cross-vm-http` web-vm bed). Added `UnmarshalJSON` + `inferListenTypes`
accepting all three shapes; the YAML path decodes the same three via
`cue.Value.Decode` running this method (the Matcher/PortScope self-decoding
strategy), so ONE decoder now serves both paths (R3). The redundant charly-side
`expandLibvirtListenersNode` expander is removed in the consuming superproject
cutover. Purely additive Go method — no wire-key change, no `#SchemaVersion` /
`sdk.ProtocolVersion` bump.
