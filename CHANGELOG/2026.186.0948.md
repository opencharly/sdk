# spec: settings wire types for the externalized `charly settings` command plugin

Adds `spec.SettingsRequest` + `spec.SettingsEntry` + `spec.SettingsReply` (settings_wire.go) — the
"settings" HostBuild kind envelope. The externalized settings command (candy/plugin-settings) OWNS the
get/set/list/reset/path subcommand grammar + output and asks the host to run each config-subsystem op
(Op ∈ get/set/list/reset/path) via HostBuild("settings"). The config subsystem (read/write
~/.config/charly/config.yml + the credential store + engine/runtime resolution) STAYS core; the reply
carries the resolved Value (get/path) or Entries (list) for the plugin to present.
