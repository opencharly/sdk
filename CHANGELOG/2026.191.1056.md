# 2026.191.1056 — the NoCloud seed ISO gets an ISO 9660-compliant volume label

Every charly VM boot emitted, twice:

```
xorriso : WARNING : -volid text does not comply to ISO 9660 / ECMA 119 rules
```

`resolveISOBuilder` passed `-volid cidata` at all three builder arms (xorriso
`-as mkisofs`, xorriso native, the genisoimage/mkisofs fallback). ISO 9660 / ECMA 119
d-characters are `A-Z 0-9 _` only, so a lowercase volume identifier is non-compliant.

**Non-corrupting, but not harmless.** The label was written verbatim (`file seed.iso` →
`ISO 9660 CD-ROM filesystem data 'cidata'`), so guests always booted. The cost was the
warning itself: a warning is not a pass (R1), and while it persisted the zero-warnings R10
gate was **un-meetable for every vm bed** — which trains readers to skim past bed-log
warnings, exactly how a real one hides.

**Fix.** ONE shared constant `cloudInitVolumeID = "CIDATA"` (R3 — the three arms passed
three separate literals, so a fourth arm could silently reintroduce the lowercase label).

**Why uppercase is safe — verified, not assumed.** cloud-init's NoCloud datasource searches
BOTH cases. Read inside a live Arch guest (`/usr/lib/python3.14/site-packages/cloudinit/
sources/DataSourceNoCloud.py`):

```python
label = self.ds_cfg.get("fs_label", "cidata")
...
label_list = util.find_devs_with("LABEL=%s" % label.upper())
label_list.extend(util.find_devs_with("LABEL=%s" % label.lower()))
```

A wrong label would break cloud-init on **every** VM, so this was proven on a live guest
before any code was written (RDD), rather than recalled from upstream memory.

**Check-coverage.** `cloud_init_iso_test.go` adds three tests, all of which FAIL against the
lowercase constant:

- `TestISOBuilderVolumeIDIsISOCompliant` — the identifier is uppercase and d-character-only,
  and still case-insensitively equals cloud-init's default `fs_label`.
- `TestResolveISOBuilderPassesSharedVolumeID` — the resolved builder's argv carries the shared
  constant, so a new arm cannot reintroduce a literal.
- `TestWriteSeedISO_NoXorrisoWarning` — actually invokes xorriso and requires ZERO warnings,
  then reads the PVD volume identifier back out of the built ISO.

Reverting the constant to `"cidata"` reproduces the production warning verbatim:

```
--- FAIL: TestISOBuilderVolumeIDIsISOCompliant
    cloudInitVolumeID "cidata" must be uppercase: ISO 9660 d-characters are A-Z 0-9 _ only
--- FAIL: TestWriteSeedISO_NoXorrisoWarning
    xorriso emitted a warning for volid "cidata"
```

Surfaced by the zero-warnings check of the superproject's `check-group` cutover
(`v2026.191.1015`), where it was disclosed rather than rounded to zero.
