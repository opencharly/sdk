# 2026.197.2205 — check-bed session wire seam narrows (K5-U1)

Schema support for the K5-U1 cutover: the check-bed host-session seam
(`#CheckBedRequest`/`#CheckBedReply`) narrows from a setup/teardown SESSION
envelope to the 9 genuinely host-shared helper ops the plugin's own
in-process session cannot self-serve.

## What changed

- `#CheckBedRequest.op` drops `setup`/`teardown` — the session itself
  (locks/lease/env/BedDescriptor) moved into the compiled-in `command:check`
  plugin. The surviving op set: `members-up` / `members-down` / `wait-ready`
  / `persist-overrides` / `gpu-prereq` / `libvirt-start` / `lease-acquire` /
  `lease-release` / `repo-override-pair`.
- `#CheckBedRequest` gains `node?: #Deploy` (the plugin threads its
  already-resolved bed node to every op but `libvirt-start`/
  `repo-override-pair`), `image_tag?: string` (`members-up`'s
  `bringUpMembers` arg), and `active?: bool` (`lease-release`'s
  was-the-lease-actually-active guard). It drops the now-unused `dir?`
  field — no surviving op needs it.
- `#CheckBedReply` gains `lease_active?: bool` (`lease-acquire`'s output)
  and `repo_override_pair?: string` (`repo-override-pair`'s output — a
  PAIR STRING only; the env `Setenv`/restore stays plugin-side). The
  existing BedDescriptor fields (calver/log_dir/is_vm/.../run_agent) are
  UNCHANGED — they are no longer wire-carried (the plugin assembles them
  directly in Go from the resolved-project envelope), but stay on this ONE
  generated struct so both roles share it (R3).
