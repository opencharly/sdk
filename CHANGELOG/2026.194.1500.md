The RESOLVED-project envelope — the third and final member of the config→spine artery

## What's new — one file, exactly TWO new views + a request

`sdk/schema/resolvedproject.cue` (package-less, concatenated into the spec compilation unit; NOT an
authoring kind — never in `#Node`/`#Op`, a pure generated wire/projection type like
`parsedproject.cue` / `loadedproject.cue`):

- **`#ResolvedBoxView`** — EXACTLY the 30 non-`json:"-"` fields of `buildkit.ResolvedBox`, in
  declaration order. This is the wire-safe half of what `charly box inspect` already serializes
  (`json.MarshalIndent(*ResolvedBox)`). The 6 host-only `json:"-"` compute-cache pointers
  (`DistroConfig`/`DistroDef`/`BuilderConfig`/`InitSystem`/`InitDef`/`CandyCaps`) are EXCLUDED by
  design — they are re-derivable by a resolving plugin, never wire data.
  - **Correction vs the spike verdict (recorded delta):** `InitSystem` is one of the 6 `json:"-"`
    caches, so it is DROPPED. The spike's THROWAWAY draft `ResolvedBoxView` struct wrongly listed
    `InitSystem string`; the live `box inspect` path (`main.go` `InspectCmd` →
    `json.MarshalIndent(*ResolvedBox)`) never emits it, so the live tree wins over the draft (the
    draft was scaffolding, not spec). The completeness test's no-cache-leak arm guards it.
- **`#CandyView`** — the resolved candy GRAPH node: identity + dep-graph (bare-ref form) + provides +
  ports/services (the exported surface of the runtime `Candy`). NOT the candy BUILD model
  (plan-steps / package-format sections — that is the CandyModel / S-CM concern, K3-D), distinct by
  design.
- **`#ResolvedProject`** = `{version, boxes: map[string]#ResolvedBoxView, candies: map[string]#CandyView,
  deploy: map[string]*#Deploy}` — the two new views composed with the EXISTING `#Deploy` recursive
  tree (`uf.Bundle` verbatim). **Deferred, planned additive members** (not omissions): the future
  `#ProvidesView` (the bundle's `provides:` map) and `#SidecarView` (the sidecar-template library),
  each added when its first consumer's K5 unit lands — no consumer in this unit reads them, so the
  envelope ships with exactly the two new views + the deploy tree.
- **`#ResolvedProjectRequest`** = `{dir, include_disabled}` — the `resolved-project` HostBuild request
  (CUE-sourced per SDD; not a hand-written wire struct).

Regenerated via `task cue:gen`: `spec/cue_types_gen.go` grows the four Go structs (`ResolvedBoxView` /
