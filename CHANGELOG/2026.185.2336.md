# sdk.OpModifierZero + sdk.RequireModifiers — collapse the 10 copy-pasted modifierZero funcs

The 10 live-verb plugins (cdp/vnc/wl/dbus/mcp/record/spice/adb/appium/kube) each hand-rolled a
`modifierZero(op, name)` func mapping a modifier wire-name to its spec.Op field's zero-check, plus a
per-plugin required-modifier loop. sdk.OpModifierZero(op, name) replaces all 10 with ONE generic
implementation (R3): reflection over spec.Op's yaml tags, so a modifier name ("tab", "key", "x2", …)
resolves to the field carrying that tag (op.Tab, op.KeyName, op.X2) and IsZero handles every field
type. sdk.RequireModifiers(method, op, required) binds OpModifierZero into the existing
CheckRequiredModifiers loop, so a plugin keeps ONLY its requiredModifiers map. TestOpModifierZeroMatchesFields
is the differential-equivalence gate: it asserts every modifier name maps to a real yaml tag and that
OpModifierZero detects zero/non-zero correctly — authorizing the replacement. CheckRequiredModifiers
stays for a verb whose zero-semantics genuinely differ from plain reflection.
