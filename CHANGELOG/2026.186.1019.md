# kit: generic yaml.Node utilities (SetByDotPath / MappingChild) for the candy command plugin

Adds `kit.SetByDotPath` + `kit.MappingChild` (yaml.go) — generic comment-preserving yaml.Node
utilities, moved out of charly core (was charly/yaml_setter.go + charly/scaffold_project.go's
mappingChild). The externalized `charly candy` command plugin (candy/plugin-candy) OWNS its entire
yaml.Node manifest mutation and uses these shared utilities directly — no HostBuild seam, no core candy
logic. `charly box set` + `charly box scaffold` (core) use the SAME kit utilities, so there is ONE
copy (R3). Tested: yaml_test.go (comment preservation, list values, intermediate-mapping creation,
scalar-descent error).
