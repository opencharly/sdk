# 2026.189.0026 — RetentionReply gains dangling/staging reclaim fields

Additive wire extension for the `retention` HostBuild reply
(`spec.RetentionReply`): `DanglingIDs` (untagged charly-built images) and
`StagingDirs` (dead buildah `/var/tmp` staging dirs) — the two reclaim
categories no existing `charly clean` verb covered, surfaced when a multi-hour
concurrent bed roster exhausted host disk (96%, 261 GB reclaimable in dangling
intermediates + ENOSPC-killed commit staging). Consumed by the compiled-in
`candy/plugin-clean` command and the core `charly/retention.go` reapers
(`pruneDanglingCharlyImages` / `pruneBuildahStaging`, both live-build-guarded).
Proto/CalVer note: JSON-only wire fields, additive → no `#SchemaVersion` bump,
no `sdk.ProtocolVersion` bump (zero-value-safe for older peers).
