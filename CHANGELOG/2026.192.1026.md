# 2026.192.1026 — the refs git primitives + the swappable fetch-backend seam land in sdk/kit (P7)

The sdk half of P7 (the remote-repo FETCH BACKEND becomes a plugin). The remote-repo GIT primitives +
cache LOCATION helpers move from charly core into `sdk/kit`, and a new `kit.RefsDownloader` seam makes the
cache-miss DOWNLOAD swappable — so both core (`reconcile`, the collection walk, bed setup) and the
compiled-in `candy/plugin-refs` fetch backend compose the SAME code (a plugin candy imports only sdk).

- **`sdk/kit/refs_git.go`** — `GitResolveRef`/`GitClone`/`RepoGitURL`/`DownloadRepo`/`DiscoverRemoteCandy`/
  `GitDefaultBranch`/`GitLatestTag`/`CompareSemver` (exported from the former unexported `compareSemver`) +
  the internal helpers. Pure git-exec + string parse; `DefaultCandyDir` was already in kit.
- **`sdk/kit/refs_cache.go`** — `RepoCacheDir`/`RepoCachePath`/`IsRepoCached`.
- **`sdk/kit/refs_downloader.go`** — the `RefsDownloader` interface (`Download(repoPath, version) → cachePath`)
  + `DefaultDownloader` (git backend, delegating to `DownloadRepo`). The swappable fetch-backend seam,
  mirroring the loader's `DocParser`: the host dispatches every cache-miss download through it, so an
  alternative refs plugin (OCI/S3) can serve a different backend. Only the DOWNLOAD is pluggable — the host
  keeps the fetch orchestration (override / cache-hit / post-fetch migrate).
- **`sdk/schema/candy.cue`** — `#PluginCapability` admits the `refs:` provider class.
- **`sdk/kit/refs_git_test.go`** — the moved unit tests.

Consumer (superproject): `charly/refs_git.go` DELETED; `candy/plugin-refs` (compiled-in) serves the git
backend; core's `EnsureRepoDownloaded` dispatches the cache-miss download through the registered
`RefsDownloader`. Behaviour-neutral (the default backend is the same git fetch).
