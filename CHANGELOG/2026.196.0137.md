# 2026.196.9999 — K4-B — the deploy-compile seam types + OpCompile + PlanFromView

K4-B (the sdk contract half of the deploy-compile-slice move): the wire types + the re-materialization half
of the round-trip that move the `charly bundle add` deploy COMPILE slice out of `charly/` core into
`candy/plugin-bundle`. The host computes the per-node selection (resolved box projected to a
`spec.ResolvedBoxView` + the FINAL pruned candy order + the host-side `deploykit.HostContext`) and Invokes the
bundle plugin's new `OpCompile` with a `spec.DeployCompileRequest`; the plugin re-hydrates the
resolved-project envelope via `HostBuild("resolved-project")`, re-hydrates the box + each candy model, loops
`deploykit.BuildDeployPlan` over the order, projects each plan to its `InstallPlanView` wire form, and returns
`[]InstallPlanView`; the host re-materializes `[]*InstallPlan` from the views via `deploykit.PlanFromView`.

## What landed in the sdk contract

- **`sdk/ops.go`** — added `OpCompile = "compile"` (a generic action selector, never a provider word — the
  F11 uniform-API gate `TestNoSinglePluginAPISurface` forbids a provider word on that surface;
  `command:bundle` is the sole first consumer). Documented at the constant.
- **`sdk/deploykit/plan.go`** — added `PlanFromView(v spec.InstallPlanView) (*spec.InstallPlan, error)`, the
  re-materialization half of the round-trip (the reverse of `WireView`), via `stepsFromView(v.Steps)`. This is
  what the host calls to turn the plugin's `[]InstallPlanView` reply back into rich in-core `*InstallPlan`s.
- **`sdk/schema/seam.cue`** (+39 lines) — CUE defs for `DeployCompileRequest` / `DeployCompileReply` /
  `ResolvedProjectRequest` (the wire carriers for the compile seam). `resolvedproject.cue` UNCHANGED.
- **`sdk/spec/cue_types_gen.go`** (+45 lines, generated by `task cue:gen`) — the generated Go for the three
  new seam types. Reproducibility-gated by `TestGenReproducible`; regeneration on a clean tree is a NO-OP.
