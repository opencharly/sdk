# Sidecar resolve wire types + opaque Deploy.Sidecar (Cutover D)

Support the sidecar de-type: candy/plugin-sidecar now owns all sidecar business
logic via an `OpResolve` leg, and the kernel stores sidecar defs opaquely.

- `spec/deploy_wire.go`: add the `OpResolve` envelope — `SidecarResolveInput`
  (opaque embedded/project/deploy def layers + CLI env + box/instance),
  `SidecarResolveReply` (resolved sidecars + app-only env + routed overrides to
  persist), and the generation-ready `ResolvedSidecar` / `ResolvedSidecarSecret`
  / `ResolvedSidecarVolume`. Add `RawBody = json.RawMessage`, a same-package alias
  so a `@go(type=map[string]RawBody)` override renders an opaque map with no
  mis-imported qualified `json.RawMessage`.
- `schema/deploy.cue`: `Deploy.sidecar` gains `@go(Sidecar,type=map[string]RawBody)`
  — CUE STILL validates each per-deploy override against `#Sidecar`, but the Go
  field is opaque (`spec.Deploy.Sidecar map[string]RawBody`). Regenerated.

Pure codegen/type change — the authored wire format is untouched, so NO
`#SchemaVersion` bump / migration entry.
