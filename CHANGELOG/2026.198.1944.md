# 2026.198.1944 — W9/K1 loader wave: CandyReader/CandyModel/CandyView + loaderkit.ScanCandy

The sdk producer leg of the W9/K1 loader-cone core-minimization wave (consumer:
opencharly/charly `feat/w9-loader-cone`, HELD until this merges — the consumer
repoints its `sdk` gitlink to the merged SHA per B2). Widens `deploykit`'s
candy-model vocabulary into a full `spec.CandyReader` interface contract and
ports the candy-manifest scan pipeline into `loaderkit`, so charly's concrete
`type Candy struct` can retype to an interface backed by CUE-schema-sourced
halves.

## CandyReader / CandyModel / CandyView (`spec`, `deploykit`)

- `spec/candy_reader.go` (new): the `CandyReader` interface contract — every
  accessor a consumer needs off a candy (build plan/package/env via
  `CandyModel`, identity/graph via `CandyView`), read-only by design once
  wrapped.
- `spec/candy_ref_entry.go` (new): `CandyRefEntry{Raw, Resolved}` (the rich
  pre-qualification ref form) + `CandyRefs{Require, IncludedCandy, BakePlugin}`
  (the mutable pipeline-state triple) + `FinalizeCandyRefs(m, v, refs)` — the
  ONE step that bare-strings refs onto the wire (`CandyView.Require`/
  `.IncludedCandy`, `type CandyRef = string`), correct for graph-topology-only
  consumers, relocated here from `loaderkit` (this cutover) since it's
  consumed by both `deploykit` and `loaderkit`.
- `deploykit/spec_candy_adapter.go` (widened): `specCandyAdapter` — the
  unexported wrapper `NewSpecCandyModel(m, v)` produces — now implements the
  FULL `CandyReader` contract, reading `CandyModel`/`CandyView` fields the CUE
  schema authors. `RawCandy() (CandyModel, CandyView)` is a structural-
  type-assertion escape hatch (not a `CandyReader` member) for the few
  host-side consumers that need the raw pair back.
- `deploykit/spec_candy_adapter.go` fix: `Port()` was emitting a lossy
  "proto:port" PREFIX for any non-empty protocol; corrected to the pre-move
  convention (bare port number, `"N/udp"` suffix ONLY for udp) — found via a
  consumer-side port-collection test sweep.
- `deploykit/hooks_collect.go` (new) + `deploykit/security.go` (widened):
  `CollectHooks`/`CollectSecurity` read the new `CandyModel.Security`/`.Hook`
  fields.

## `loaderkit.ScanCandy` — the candy-manifest scan pipeline

- `loaderkit/scan_candy.go` (new, 402 LOC): ports the pre-move
  `scanCandy`/`populateCandyFromYAML`/`derivePackageSections` logic from
  `charly/layers.go`, producing `spec.ScannedCandy{Model, View, Refs}` — the
  mutable pre-wrap triple the consumer's scan→fetch→qualify→arbitrate
  pipeline carries before wrapping into a `CandyReader`.
- `loaderkit/repo_identity.go` (new): repo-identity cycle-break for the
  namespaced-import loader (relocated from `charly/ns_identity.go` +
  `charly/main_repo.go`'s `normalizeRepoSpec`) + `NormalizeRepoSpec` (the
  `--repo` spec normalizer) — pure fs/git/yaml logic, no `*Config`/`*Candy`
  dependency.
- `spec/namespace_ref.go` (new): the generic hierarchical-namespace-ref vocab
  (`SplitNamespaceRef`/`LeafName`) — a wide, non-loader-cone consumer set.
- `spec/ref_parse.go` (new): shared remote-ref string parsing, legal for any
  charly package to import.

## CUE schema (SDD) — additive only

- `schema/candymodel.cue`: `#CandyModel` gains `security`, `hook`, `artifact`,
  `requires_capability`, `capability`, `secret`, `port`, `bake_plugin` — the
  remaining fields the 42-file consumer-side `CandyReader` repoint needed,
  mirroring `candy.cue`'s already-authored shapes exactly.
- `schema/resolvedproject.cue`: `#ResolvedBoxView`'s candy view gains
  `init_systems` (the per-init-system trigger map — `has_init` alone is one
  aggregate bool, insufficient for a per-init `HasInit(name)` lookup /
  render-time `EmitInitFragmentStages`).
- All additive on existing defs; generated via `task cue:gen`, zero
  hand-written wire structs, zero new disjunctions. `spec.CandyRefEntry`/
  `CandyRefs` and `RawCandy()`'s raw pair are legitimate hand-written NON-wire
  internal shapes (pipeline state / a structural-assertion escape hatch), the
  documented SDD exception category.
