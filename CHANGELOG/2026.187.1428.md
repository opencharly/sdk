# Init resolve wire types (Cutover F)

Support the init de-type: candy/plugin-init resolves the init-system vocabulary into
generation-ready envelopes the kernel consumes without importing spec.Init.

- `spec/init_wire.go` (hand-written): the service-render leg (`ServiceRenderContext`,
  `RenderedService`, `KeyValue`, `ServiceRenderInput/Reply`) + the config leg
  (`ResolvedInit` — the build/label/entrypoint value envelope with `Raw` for the
  render, `InitResolveInput/Reply`) + the discriminated `InitResolveRequest`
  (Render | Config).

No CUE-schema change (the authored `#Init` shape is untouched) — no `#SchemaVersion`
bump, no gen-repro drift.
