# 2026.193.2345 — P11: the pod config-WRITE mechanism + the config-write seam

2026-07-12

The sdk half of P11 (pod config-WRITE externalization). charly core's pod config-write
mechanism moved into `sdk/deploykit`, and the host↔plugin config-write seam +
`OpConfigWrite` selector are added, so the `deploy:pod` plugin can render + write a pod's
quadlet files from a host-resolved `QuadletConfig`.

## deploykit — the config-WRITE mechanism (relocated from charly core)

- `quadlet.go`: `GenerateQuadlet` + its `[Unit]`/`[Container]`/`[Service]`/`[Install]`
  emitters; the resolved-runtime input types `QuadletConfig` / `CollectedSecret` /
  `ResolvedBindMount` / the host-adapted `ResolvedSidecar`; the pure helpers
  `ResolveWorkingDir` / `BareVolumeName` / `LocalizePort` / `IpcModeBlocksShmSize` /
  `SchemeTarget` / `TailscaleFlag` / `IsTCPFamily` / `TunnelServiceFilename` /
  `NormalizeCgroupSize` / `FormatCPUQuota`; and `GenerateTunnelUnit(cfg, cfgPath)` (the
  cloudflare companion-unit emitter — it now takes the cloudflared config path as a
  parameter, so the resolution helper `tunnelConfigPath` can stay in charly core).
- `quadlet_pod.go`: the pod + sidecar content generators `GeneratePodQuadlet` /
  `GenerateSidecarQuadlet` (+ the internal `hasTailscaleSidecar` / `sortedSidecarEnv` /
  `isSidecarEnvQuotable`).
- `deploy_volume.go`: the enc-coupled volume resolver `ResolveVolumeBacking` +
  `resolveVolumeHostPath` + the pure enc-path cluster `EncryptedVolumeName` /
  `EncryptedCipherDir` / `EncryptedPlainDir` (folds with the enc-model cutover, as the P13
  file header earmarked).
- All are PURE (no host I/O): a host resolves the inputs and a compiled-in-or-out-of-process
  plugin renders. Byte-identical to the former in-core generators (the goldens moved with
  the engine — the ~21 emission tests + the pod/sidecar/tunnel + volume-backing/enc-path
  tests all live in deploykit now).

## The config-write seam + selector

- `schema/seam.cue`: `#PodConfigWriteRequest` (the resolved `QuadletConfig` as an opaque
  `RawBody` envelope — the VmJSON pattern — + the host-computed absolute paths:
  `container_path` required; `pod_path` / `sidecar_paths` / `tunnel_path` /
  `cloudflared_cfg_path` optional, a set optional path being the host's signal to write that
  file kind) + `#PodConfigWriteReply{written_paths}`. Generated faithfully by `task cue:gen`
  (`RawBody`, `map[string]string`, `[]string`); reproducibility gate green.
- `ops.go`: `OpConfigWrite = "config-write"` — the pod config-write Op the deploy:pod plugin
  serves; `charly config` (host) Invokes it after resolving the QuadletConfig + the paths.

## The tunnel wire-type repair (Task A)

- `schema/tunnel.cue` + `spec/tunnel_methods.go`: `TunnelConfig` / `TunnelPort` (+
  `TunnelPort.Backend()`) are CUE-sourced now (the SDD wire-type mandate); the former
  hand-written mirror in charly core was deleted and aliased to `spec`.
