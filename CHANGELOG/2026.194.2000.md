# 2026.194.2000 — K1: the unified-config loader WALK relocates to sdk/loaderkit

2026-07-13

The kind-blind ORCHESTRATION half of charly's `LoadUnified` — the `import:` queue walk, the
`discover:` walk, the namespaced-import mounts, and the per-document parse — relocates OUT of
charly core into `sdk/loaderkit` as a new `Walk` function returning a generic
`spec.LoadedProject` envelope. The host keeps (and correctly retains) the registry-coupled
MATERIALIZE + root-wins MERGE; it now replays them over the returned envelope. This is the K1
step of the v2 plugin-host program: the loader becomes a plugin-servable, kind-blind mechanism
(boundary law clause D — it consults host-threaded DATA + six kind-blind host seams, never the
provider registry directly), so a future OpLoad-envelope unit (K5) can serve the whole project
load through candy/plugin-loader and delete charly core's last direct loaderkit import.

## New

- **`spec.LoadedProject` envelope family** (CUE-sourced, `schema/loadedproject.cue`): the WALK
  result — `docs` (root file + flat imports, in walk order), `discovered` (the `discover:`
  manifests), `namespaces` (mounted namespace subtrees), and a per-walk `id`. `#NamespaceMount`
  carries either a DEFINITION (inline `project`) or a symbolic REFERENCE (`ref`/`refID`) for a
  repo-identity cycle-break / diamond re-import, so the host materialize can share the same
  materialized project by id (register-before-recurse) and preserve the original loader's
  pointer identity across the intentional mutual import (main↔cachyos). `#LoadedDoc.directives`
  is the RAW reserved-directive mapping as YAML bytes (the host replays the original decode
  exactly, honoring the custom import/discover unmarshalers). Self-recursion is modeled as
  `[]*NamespaceMount` (the go-skill-proven gengotypes shape).
- **`spec.Threaded` + `spec.DocParser`** (`spec/loader_seam.go`): the loader-seam CONTRACT types
  moved here from `sdk/loaderkit` so BOTH the host and the loader plugin reference them without
  importing each other. `loaderkit.ParseDoc` now takes `spec.Threaded`.
- **`loaderkit.Walk(rootDir, rootData, rootIdentity, WalkSeams)`** (`loaderkit/walk.go`,
  `discover.go`): the relocated orchestration. `WalkSeams` is the six kind-blind host callbacks
  — `Parser` (spec.DocParser), `Boundary` (prescan + connect-declared-kind-plugins),
  `Threaded` (registry snapshot), `ResolveRef` (remote fetch + cache), `GateDoc` (#NodeDoc
  gate), `RepoIdentity` (cycle-break). Walk does NO materialize, NO merge, NO registry access.
- **`sdk/kit` loader primitives** (`kit/loader_directives.go`, `loader_discover.go`,
  `loader_classify.go`): the kind-blind config-directive types + fs walk shared by BOTH charly
  core AND loaderkit (R3, one home) — `ImportList`/`ImportEntry`, `DiscoverConfig`/`ScanSpec`,
  `AnchorScanSpecs`, `ClassifyDoc`/`DocShape`, `FindEntityDirs`/`DiscoverSkipDir`.

## Removed

- `loaderkit.DefaultParser` (the in-loaderkit fallback DocParser) — DELETED. The compiled-in
  candy/plugin-loader is the sole parser, registered at init before any load; the host FATALs on
  a nil parser (never a silent fallback).

Regeneration on a clean tree is a no-op (`TestGenReproducible` green). See the superproject
CHANGELOG for the core-side relocation (loader_driver.go + the deleted orchestration).
