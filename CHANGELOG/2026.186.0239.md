# M4a: substrate-lifecycle wire types for the externalized pod deploy lifecycle

Added the host↔plugin envelope for the pod deploy substrate lifecycle Ops, so the compiled-in pod
lifecycle can move to candy/plugin-deploy-pod (M4): LifecycleOpts (the serializable EmitOpts subset —
ParentExec/ParentNode re-attach host-side, never serialized), HostEnv {CharlyBin, Home} (the host
charly path handed to the plugin, whose own os.Executable() is the plugin binary), PrepareVenueReply
{Venue, State, Notes} (replaces the bare VenueDescriptor OpPrepareVenue reply), PostTeardownReply
{RemoveEntries}, and CliRequest/CliReply (the generic "cli" host-builder envelope — the plugin asks
the host to run a `charly <argv>` subcommand). All ride Provider.Invoke JSON; no new proto RPC.
