# CheckContextService.ResolveClusterContext — k8s cluster-context reverse-leg (H part 2d)

Add a class-generic `ResolveClusterContext(cluster)→context` RPC so a cluster-probing verb
(kube) maps a charly k8s cluster-profile name to its kubeconfig context itself, instead of the
host rewriting the op param — the host owns the project loader (findK8sSpec) the out-of-process
plugin cannot reach.

- `proto/plugin.proto`: `rpc ResolveClusterContext(ResolveClusterContextRequest{cluster})
  returns (ResolveClusterContextReply{context,error})` on `CheckContextService` (regenerated
  stubs). Concept-named (not verb-named) → class-generic, never a per-verb RPC (Uniform API).
- `kit.CheckContext`: add `ResolveClusterContext(ctx, cluster) (context, error)`.
- `checkverb.go`: `sdkCheckContext.ResolveClusterContext`.

No CUE-schema change.
