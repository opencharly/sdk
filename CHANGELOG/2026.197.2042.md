# 2026.197.2230 — K3 opener: PackageNames/PackageItemsFromStrings → sdk/spec

Cross-repo landing, sdk half. The `charly` repo's K3 core-minimization cutover
(build-engine lane) dissolves `charly/calamares_types.go` as part of its
ZERO-ALIASES/R3 alias-and-dead-code sweep. `calamares_types.go` carried two
pure functions operating on `spec.PackageItem` that had no business staying in
core once the type itself already lived in `sdk/spec` — they belong alongside
their type, reachable by every consumer (host and out-of-process plugins
alike), not duplicated in charly core.

## Change

`spec/union_types.go` — added, immediately after the `PackageItem` type
definition it operates on:

- `PackageNames(items []PackageItem) []string` — returns just the names from
  a `PackageItem` list, in order.
- `PackageItemsFromStrings(names []string) []PackageItem` — constructs a
  `PackageItem` slice from bare names (used by the migrator collapsing legacy
  format sections that only carried `packages: [name1, name2]`).

Updated the `PackageItem` doc-comment, which named `calamares_types.go` as the
functions' historical home — now stale since the file is being deleted in the
consuming `charly` PR; removed the reference (R1).

`spec.PackageNames`/`spec.PackageItemsFromStrings` and deletes
