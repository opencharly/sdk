# CheckContextService.ResolveGraphicsEndpoint — VM graphics reverse-leg (H part 2b)

Add a second class-generic reverse-leg to the check-verb reverse channel so a VM graphics verb
(vnc/spice) resolves its OWN dialable endpoint instead of the host preresolving it per-verb.

- `proto/plugin.proto`: `rpc ResolveGraphicsEndpoint(ResolveGraphicsEndpointRequest{kind})
  returns (ResolveGraphicsEndpointReply{addr,socket,password,skip,skip_message,error})` on
  `CheckContextService` (regenerated stubs). Parameterized by kind ("vnc"|"spice") — shared by
  both verbs, never a per-verb RPC (the Uniform API Invariant).
- `kit`: `CheckContext.ResolveGraphicsEndpoint(ctx, kind) (GraphicsEndpoint, error)` + the
  `GraphicsEndpoint{Addr,Socket,Password,Skip,SkipMessage}` result struct.
- `checkverb.go`: `sdkCheckContext.ResolveGraphicsEndpoint` (calls the RPC, maps the reply).

No CUE-schema change.
