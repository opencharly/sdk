# spec: feature wire types for the externalized charly feature command plugin

Adds `spec.FeatureRequest` + `spec.FeatureStep` + `spec.FeatureEntity` + `spec.FeatureReply`
(feature_wire.go) — the "feature" HostBuild kind envelope. The externalized `charly feature` command
(candy/plugin-feature) OWNS the list/pending/validate grammar + output; the unified LOADER (LoadConfig /
ScanCandy — the kernel), the Step plan model, and validatePlanSteps (shared with `charly box validate`,
R3) stay core and enumerate every entity's plan into this plain DATA for the plugin to format.
