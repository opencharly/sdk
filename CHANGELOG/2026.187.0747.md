# Schema compaction — the `#Op` diet, env map unification, capability primaries

Schema `2026.186.2323` (floor stays `2026.174.1100`). Breaking format change;
consumers migrate via the superproject's `charly migrate` (compact-node-form).

- `#Op` dieted 126 → ~35 fields: the 11 live verbs' method discriminators +
  exclusive modifiers moved into their plugins' input schemas; the never-authored
  harness surface (summarize/kill/signal/aggregation/capture/parallel) deleted;
  `plugin`/`plugin_input`/`command` are INTERNAL-ONLY (excluded from
  `AuthoringVerbs` via schemagen's `opRuntimeDerivedFields`).
- `#OpVerb` = mkdir|copy|write|link|download|setcap|build|plugin.
- `env` unified to the map form on every kind (`#EnvVar` deleted;
  `BoxMetadata.Env` stays `[]string` as the label wire format).
- `#DeployShellOverlay` factored onto shared `#Shell`; Calamares defs
  (`target.cue`/`module.cue`/`group.cue`) deleted.
- `ProvidedCapability` gained `Primary` (proto field 10): a class-verb
  capability's scalar-sugar primary input field; `#Plugin` gained the manifest
  `primary?: {[string]: string}` map (the pre-connect prescan source).
- `#PluginCapability` gained the `command` class + underscore words.
- `BuildCapabilities` accepts a nil schema FS for an input-less plugin (the
  stub-schema requirement is gone).
- `#Candy.engine` retained with a written justification; vestigial live-verb
  method string aliases removed from `spec/scalar_aliases.go`; `spec.DataKeys`
  and the schemagen DataKeys emission deleted with the child-node grammar.
