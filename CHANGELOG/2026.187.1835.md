# VM resolve wire types (Cutover L — the vm substrate-value de-type)

Support de-typing the `vm` substrate value: candy/plugin-substrate resolves an
authored `vm:` template into a `ResolvedVm` the kernel's vm build/deploy consumers
read without importing the concrete `spec.Vm`.

- `spec/vm_wire.go` (hand-written): `ResolvedVm` — mirrors `spec.Vm`'s 14 fields +
  `Raw` (no methods on `spec.Vm`, so a pure field mirror).
- `spec/substrate_template_wire.go`: `VmResolveInput` / `VmResolveReply` + a `Vm` arm
  on `SubstrateTemplateResolveRequest` (alongside Local/Android/Pod/K8s).
- `spec/deploy_wire.go`: `LifecyclePrepareInput.VM` is now `*ResolvedVm` (was
  `*VmSpec`) — the vm deploy plugin reads the resolved envelope over the wire.
- `vmshared/spec_aliases.go`: `VmSpec = spec.ResolvedVm` (was `spec.VmSpec`) — this
  ONE alias flip moves every kernel vm consumer (26 of 27) + the vmshared
  render (qemu/cloud-init/libvirt) to the resolve envelope.

No CUE-schema change (`#Vm` untouched) — no `#SchemaVersion` bump, no gen-repro drift.
