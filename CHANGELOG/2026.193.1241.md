# 2026.193.1241 — scaffold ENGINE relocated to sdk/kit (P15 sub-unit, sdk half)

The project/candy/box authoring ENGINE — the `charly box new` machinery — moves from charly core
into `sdk/kit` so BOTH the core CLI (`charly box new`) AND the Wave-2 `command:box` plugin drive ONE
shared library (R3). A plugin imports only the sdk, so the create-side engine cannot stay in core if
a plugin must call it. This is the sdk (producer) half; the core rewire + `scaffold.go` deletion ride
the paired superproject PR, which bumps the sdk pointer to this commit first.

## kit/scaffold.go (new)

- **`ScaffoldCandy(dir, name, calver)`** — creates `candy/<name>/charly.yml` (the ADE-minimal seed:
  a `description:` + one deterministic `check:` step). `calver` stamps the mandatory `version:`.
- **`ScaffoldProject(dir)`** — seeds a fresh project (`charly.yml` + `box/` + `candy/` + `.gitignore`),
  stamping the schema version from `LatestSchemaVersion()`. Refuses to clobber an existing project.
- **`AddBox(dir, name, base, layers)`** — writes `box/<name>/charly.yml` in the node-form image shape
  (`<name>: {candy: {base: …, candy: [layers…]}}`).
- The seed templates (`scaffoldCharlyYAML` / `scaffoldGitignore`).

Pure fs + yaml.v3; the only charly-layout knowledge is the kit layout constants
(`UnifiedFileName`/`DefaultCandyDir`/`DefaultBoxDir`) kit already owns, plus `FileExists` +
`LatestSchemaVersion`. Wall-clock stays out of kit: `ScaffoldCandy` takes the candy `version:` CalVer
as a param (the caller passes charly's `ComputeCalVer()`), mirroring the migrate helpers. The
functions do the filesystem work and return; the CALLER owns user-facing output (kit stays
presentation-free).

## kit/yaml.go

- **`SaveYAMLNodeFile(path, root)`** — the comment-preserving marshal-to-file step, now an exported
  kit helper shared by `kit.AddBox` and charly core's `box add-candy`/`rm-candy` verbs (ONE home, R3).

## Tests

`kit/scaffold_test.go` — `ScaffoldCandy` (create + don't-clobber), `ScaffoldProject` (seed + guard),
`AddBox` (node-form + don't-clobber). All pass. The superproject keeps its integration tests
(add-candy/rm-candy round-trips) rewired to call `kit.ScaffoldProject`/`kit.AddBox`.

Additive — nothing removed from the sdk. Verified: `go test ./kit/` green; the superproject builds +
`charly box new project/candy/box` smoke passes against this kit engine.
