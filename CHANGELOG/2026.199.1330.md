# 2026.199.1330 — CUE wire generation and generic provider channels

- Make `protocol/schema/*.cue` authoritative for the protobuf IDL, services,
  streaming flags, and generated Go stubs. `task wire:gen` is reproducible and
  `proto:gen` remains an alias.
- Add the bidirectional `Provider.Channel` transport, sequence/replay gates, and
  generic CUE target routes with local-exec and gRPC-over-SSH stdio dialers.
- Generate agent, team, incident/RCA/recovery, target, and terminal records from
  CUE, plus the `ai.opencharly.agent_provide` capability label.
- Add an RCA-gated recovery state machine that permits only an explicit
  emergency abort before RCA completion.
- Add daemon-free durable agent storage with cross-process locking and atomic
  idempotency reservations, team delegation validation, federation records,
  and a generated `ProcessLaunch` contract that preserves argv, environment,
  and working directory through local, SSH, and arbitrarily nested executors.
- Enforce the authoritative CUE contracts at every durable agent-store
  mutation and read ingress. Strict typed JSON decoding rejects unknown fields,
  trailing values, invalid base64/integer representations, malformed UUIDv7
  identifiers, invalid enums, and empty idempotency keys. Persisted records
  must match their filename identity, while JSONL records must match their run
  and sequence rules. Invalid identifiers are rejected instead of collapsing
  to a shared filename, preventing overwrite and cross-read aliasing.
- Leave protobuf field 12 available for the concurrent command-subcommand
  catalog and move `ProvidedCapability.command_model_json` to field 13 in the
  authoritative CUE protocol schema, with generated-descriptor and
  wire-round-trip coverage.
- Keep target working directories and environment on their owning side of a
  process transport: local exec applies them to the endpoint child, while SSH
  renders them only into the remote command. Command-factory environment is
  preserved and target values overlay it deterministically. Real-OpenSSH
  coverage verifies the working directory and environment at the remote
  endpoint without leaking them to the local carrier. Direct and nested SSH
  process-executor coverage preserves argv, pipes, working directory,
  environment, and RFC 4254 exit status, with one shared zero-warning log-level
  option on every nested SSH/scp surface.
- Build every native local package from an isolated, mode-preserving source
  copy while exposing the authored directory only as an explicit read-only
  template input. Package tools can no longer rewrite tracked definitions or
  leave build trees in pac, rpm, or deb source directories.
