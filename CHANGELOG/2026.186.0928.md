# spec: retention wire types for the externalized `charly clean` command plugin

Adds `spec.RetentionRequest` + `spec.RetentionReply` (clean_wire.go) — the "retention" HostBuild kind
envelope. The externalized clean command (candy/plugin-clean) OWNS the command (flag grammar, category
orchestration, output, the local pkg/arch makepkg sweep) and asks the host to run the SHARED retention
engine (image-tag / build-candy / check-run pruning — also called by `charly box build`, `charly check
run`, `charly box list tags`, so it stays core) via HostBuild("retention"). The engine resolution
(ResolveRuntime + keep_images/keep_check_runs defaults) happens host-side; the reply carries the removed
refs/dirs/paths + effective counts for the plugin to present.
