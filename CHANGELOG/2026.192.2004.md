# 2026.192.2004 — plugin-declared DeployTraits unify into the descent descriptor (P9 sdk half)

The sdk half of P9 (traits/descent de-branching). A substrate's venue behaviour becomes PLUGIN-DECLARED
data — a `#DeployTraits` the substrate plugin advertises over Describe — and it is unified INTO the EXISTING
loader-stamped descent descriptor (R3: ONE mechanism, no parallel word→transport table). The kernel/deploy
Mechanism consults the stamped trait off `node.Descent`, never a `switch` on the substrate kind word (the
boundary law's incomplete-seam).

- **proto** (`proto/plugin.proto` + regenerated `plugin.pb.go`): new `DeployTraits {venue, image_backed,
  image_context, machine_venue, exclusive_venue, leaf_only}` message + `deploy_traits = 11` on
  `ProvidedCapability` — the wire channel a substrate plugin declares its venue behaviour over.
- **CUE** (`schema/deploy.cue` → regenerated `spec/cue_types_gen.go`): `#DeployTraits` def, embedded into
  `#DescentDescriptor` (which keeps `transport`/`host_rooted` as the loader-DERIVED nesting view).
- **`schema.go`**: `ProvidedCapability.DeployTraits *spec.DeployTraits`; `BuildCapabilities` marshals it.
- **`kit/descent.go`**: the `descentForTarget` word→transport SWITCH is DELETED. `StampDescent(node,
  traitsFor)` stamps the declared traits; `DescentFromTraits(traits)` derives transport/host_rooted
  GENERICALLY (leaf_only→reject, container→container-exec, ssh→ssh, nil→container-exec = the former default).
  Behaviour byte-preserved (unit test proves all former substrate words).
- **`deploykit/deploy_tree.go`**: the DEAD post-order teardown walk (`WalkDeploymentTreePostOrder` +
  `DeriveChildExecutor` + `containerChildExecutor` + `containerNameForNode`) — superseded by the trait-based
  flat-path chain (`AppendHopForFlatPath`, deploy_chain.go) for BOTH deploy and teardown — is DELETED,
  removing its `switch node.Target` child-executor derivation (P9, S2). `VmChildExecutor` stays (the live
  flat-path visitor hops through it for a vm venue).
- **`deploykit/install_build.go`**: the machine-venue signal moves from a `HostContext.Target` string
  compared to `"vm"` (a dead arm) to a `MachineVenue bool` read off the trait.
- **`deploykit/deploy_state.go`**: `BedCheckLiveRefs` android-skip reads the venue trait; `findVmDeploy`
  keeps the persisted-`Target` read (the deploy state STRIPS `node.Descent` on save — "loader-DERIVED, never
  operator-authored" — so the trait is unavailable there; allowlisted in the consumer's word-switch gate,
  with a tracked follow-up to persist the descent).

Consumer (superproject): charly's deploy/check consult sites read `node.Descent` traits; candy/plugin-substrate
declares the canonical per-word trait table; a structural gate (over charly + sdk/deploykit) forbids a regrown
`node.Target == "<substrate>"` behaviour-branch. Behaviour-neutral. `go test ./...` green; `golangci-lint run`
0 issues; `gofmt`/`go vet` clean; gen reproducible.
