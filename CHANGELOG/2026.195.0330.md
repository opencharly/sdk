# 2026.195.0330 — R44 — check-box container-setup infra classification + disposable-box-executor removal (breaking!)

Half of the exit-127 check-box store-contention fix (task #44): a `charly check box` probe container's rootfs
mount + temp-`/etc/passwd` generation, racing concurrent image-build layer commits in the shared podman
store, was laundered into a check FAILURE. Adds infra-failure classification and removes the O(N)-per-step
disposable-container box executor (the superproject moves box-mode to ONE persistent container).

## Container-setup infra classification (kit/infra_classify.go, NEW)

- `ClassifyContainerInfraFailure(exit, stderr)` — a named, provenance-commented signature table: exit-125
  (podman's OWN error exit) is the PRIMARY signal; 8 stderr substrings (each commented with its
  podman/OCI/c-storage origin — "creating temporary passwd file", "unmounting container", "layer not known",
  "image not known", "database is locked", …) are the SECONDARY discriminator. A genuine in-container
  "command not found" (bash ran, command absent) matches nothing and stays an ordinary check result.
- `NestedExecutor.RunCapture` classifies at the ONE container-jump seam → returns a MARKED infra error
  (`ContainerInfraErrMarker`, mirroring `SignalKillErrMarker`) that rides each verb's existing
  `execution error: %v` into the CheckResult — zero per-verb change. `ContainerInfraError` is exported for the
  superproject's container-CREATE classification.
- `report.go`: a distinct `INFRA` step label. `eventually.go`: a container-setup infra failure is deliberately
  NOT retried (classify-only — the persistent-container box-mode removes the O(N) setup root, so a residual
  infra event is surfaced LOUDLY, never absorbed by a retry).

## Disposable-container box executor DELETED (breaking!)

`deploykit.ImageChain` (the `podman run --rm <img> bash`-per-step box executor) + `kit.JumpPodmanRun` /
`JumpDockerRun` + `NestedJump.ProbeName` + `probeNameSeq` + `NestedExecutor.reapDisposableProbe` (the
disposable-probe naming + reap machinery) are DELETED — box-mode now runs its probes inside ONE persistent
container via `podman exec` (the superproject's `checkBoxContainerChain`), O(N)→O(1) container setups. Grep
proved ZERO production consumers remained. `prepareJump` simplified to `(string, error)`; the `run`/`RunCapture`
reap `defer`s and the wrapWithJump/PutFile/Kind/String disposable-run cases removed.
