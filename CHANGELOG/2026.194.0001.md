# 2026.194.0001 — the check ENGINE (Runner) relocates into sdk/kit (P12a, sdk half)

The sdk half of the P12 check-plugin cutover, landing FIRST (cross-repo producer order). The
check-engine **Runner** — the host context the plan walk drives (`RunOne`/`RunPlan` consume it) —
relocates from charly core into `sdk/kit` as an importable, constructor-shaped library type, so any
plugin candy that runs a plan drives the SAME loop the in-core check engine uses. Behaviour-neutral:
`charly check box` is BYTE-IDENTICAL across every `--format` (text/json/tap/junit) before vs after,
including the context-skip grammar seam (parity gate, the R10 for this relocation).

- **`kit/runner.go`** (new) — `Runner`, the library form of the plan walk's host context. It
  implements `PlanContext` (the surface `RunOne`/`RunPlan` consume) and carries the shared engine
  state. It is **CONSTRUCTOR-SHAPED**: its state is unexported and its interface methods
  (`Mode`/`Distros`/`VerifyOnly`/…) would collide with same-named data fields (the exact reason the
  former core `runnerPlanContext`/`runnerCheckContext` were wrappers), so the host builds it via
  `kit.NewRunner(kit.RunnerConfig{…})` and reaches its state through methods — the genuinely
  library-shaped API a plugin gets too. `var _ PlanContext = (*Runner)(nil)` locks the contract.
  - **`PlanGrammar`** interface — the do-mode + execution-context grammar seam
    (`EffectiveDo`/`InContext(op, runtime)`/`ContextsLabel`). charly core's `VerbCatalog` impl stays
    core (`hostPlanGrammar`), injected into the Runner.
  - **`VenueResolver`** func — the `on:` per-step venue-swap seam; the host `check_venue` atom is
    injected, returning kit types, so `SwapVenue` logic lives IN `kit.Runner`.
  - **Exported accessors** (`Exec`/`Box`/`Instance`/`DialTimeout`/`HTTPClient`/`VmName`/
    `VmTargetName`/`HasRuntime`/`CandyDirs`/`CandyScanErr`) — read by charly core's thin
    `hostCheckContext` wrapper (which keeps the 12-method `kit.CheckContext` surface, its 6
    host-coupled legs — `HTTPDo` + the four `Resolve*` podman/go-libvirt/ssh ops — staying core;
    a plugin module lacks that machinery).
  - `runnerProbeNeverHangFallback` — a kit-local defensive per-probe ceiling used ONLY when the host
    leaves `cfg.ProbeTimeout` zero (the host always sets it from its readiness table). Kept in kit,
    not a core-const reference, so kit stays standalone (R3).

Consumer: the superproject repoints ~6 `NewRunner` sites + 8 `RunPlan` sites to `kit.NewRunner`/
`kit.Runner`, adds the host seams (`hostVerbResolver`/`hostPlanGrammar`/`venueResolver`/
`hostCheckContext`), and DELETES the old core `Runner` struct + adapters + `RunLive` +
`FormatResults*` (R5). It lands as its own PR after this sdk half merges + tags.
