# 2026.205.1440 — K1-unblock W3 Unit B: container VenueDescriptor + CheckEnv + CheckLoadPluginsRequest

The sdk-side companion legs for the `charly` superproject's K1-unblock wave
(check-family relocation, arms 1-4 — the check verbs moved plugin-side via
`InvokeProvider`, plus the `DescribeProvider` third branch). Three new/extended
CUE-sourced wire shapes, landed together because they were all proven live by
the same `check-pod` bed run on the charly-side branch, plus a follow-up
comment sweep.

## What changed

- **`#VenueDescriptor` gains a third round-trippable kind, `"container"`**
  (`sdk/schema/deploy.cue`, alongside `"shell"`/`"ssh"`), so the ONE
  enumerable `*NestedExecutor` shape `deploykit.ContainerChain` always
  produces (a plain `ShellExecutor{}` parent, a single `JumpPodmanExec`/
  `JumpDockerExec` hop, no `ExtraArgs`) can serialize across the
  `Executor.InvokeProvider` wire boundary and re-materialize byte-identically
  on the other side. `kit.VenueFromDescriptor`'s `"container"` case +
  `kit.ContainerChainFromDescriptor(engine, containerName)` is the shared
  construction both directions use (R3); `deploykit.ContainerChain` now
  delegates to it instead of duplicating the shape.
  `kit.DescriptorFromExecutor`'s `*NestedExecutor` arm recognizes ONLY that
  exact single-hop shape and falls through to the zero descriptor for
  anything else (a different parent, `JumpSSH`/`JumpVirshConsole`, or
  non-empty `ExtraArgs`) — deliberately not generalized to arbitrary-N-hop
  composition, which stays unsupported.
- **`#CheckEnv` (`sdk/schema/checkresult.cue`)** converges the check-verb
  invocation-context snapshot to ONE CUE-sourced shape, replacing three
  independently hand-maintained mirrors of the same wire contract: `sdk`'s
  own private `checkEnvWire` (deleted, replaced by `spec.CheckEnv`
  throughout), `charly`'s hand-written `CheckEnv` struct (becomes a type
  alias on the charly side), and a hand-mirrored struct in
  `candy/plugin-check` caught before its six-arm fan-out multiplied
  consumers. `#CheckEnv` is a superset of the two pre-existing shapes (adds
  `container_name`/`venue`); every field is optional since different callers
  populate different subsets.
- **`#CheckLoadPluginsRequest` (`sdk/schema/seam.cue`)** — the plugin-connect
  entry seam for K1-unblock arm 1 (`hostCheckRunLive`): the plugin-side live
  check-run gather (`candy/plugin-check`'s `pluginCheckRunLive`) needs to
  trigger the host's existing plugin-connect engine
  (`resolveCheckRunnerContext`) before dispatching a plan whose verb words
  may reference an out-of-process candy — connecting a plugin is the
  clause-M kernel mechanism that stays core, so it rides this thin new seam.
  Mirrors `#PodDisposableRequest`'s shape (name + optional dir).
- **R5 stale-reference sweep** (comment-only, no behavior change):
  `kit/grader.go`, `kit/tagexpr.go`, `kit/checkrun_seam.go`,
  `kit/description_merge.go`, `deploykit/description_collect.go`,
  `schema/seam.cue` — every mention of `RunCheckLive` /
  `hostCheckRunFeatureLive` / `resolveGraderAgent` / `checkLivePod` as
  still-core or host-resident now correctly names their plugin-side
  successors (`pluginRunCheckLive`/`pluginCheckRunFeatureLive`/
  `resolveAgentSpec`/`pluginCheckLivePod`, all in `candy/plugin-check`).
  `task cue:gen` regenerated `spec/cue_types_gen.go` from the edited
  `schema/seam.cue` comments — reproducibility-gated, diff is comment-only.
