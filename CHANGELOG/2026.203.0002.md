## What this leg is

The sdk-contract counterpart to the superproject's FINAL/K5 unit 6 landing
(opencharly/charly#171 — the consolidated unit 6a + unit 6b integration).
Two units' sdk work, already merged into one head (`ced6e9a` — unit 6b's
branch merged unit 6a's sdk leg mid-flight) before this PR's branch point:

- **Unit 6a** — the ephemeral-lifecycle cross-substrate seam + the generic
  discriminated deploy-entity-resolve envelope + vm check-live hardening:
  - `ops.go`: `OpEphemeralRegister`/`OpEphemeralTeardown` (the
    `command:bundle` ephemeral-lifecycle action selectors — the host
    Invokes these as the first action of an ephemeral deploy's Add and the
    last of its Del) + `EphemeralPanicMarker` (a STRING marker, wire-
    portable, that a `command:bundle` panic recovery converts to, so a
    genuine bug FAILS the whole deploy Add instead of vanishing silently —
    RCA #5).
  - `spec/hand_state_types.go`: `EphemeralRuntime.DeployAddress` — the
    CLI-addressable dotted deploy identity, distinct from the dot-sanitized
    `vm:<domain-id>` map key the entry is stored under (`VmDomainIdentity`'s
    `.`→`-` transform is lossy; `teardownChildrenRec` needs the original
    address back — RCA #2). `schema/deploy.cue`'s `#VmDeployState` gains a
    matching validation-only `ephemeral?:` sub-def (every field optional,
    tolerating legacy/partial entries) for SDD lockstep with the hand type.
  - `schema/seam.cue` (NEW): the command-time host↔plugin seam wire types
    (P10) — `#ConfigResolveRequest`/`Reply` and the generic discriminated
    entity-resolve request/reply pair the `deploy-entity-resolve` HostBuild
    seam uses (ONE envelope serving android/k8s/vm entity lookups by kind,
    instead of one HostBuild kind per substrate).
  - `deploykit/deploy_state.go`: `FindVmDeployNode`'s ambiguous-fallback
    scan hardened — new signature `(BundleNode, bool, error)`, returns an
    error on 2+ candidates instead of nondeterministically first-winning
    (Go's per-process map-iteration randomization made this a real,
    intermittent misroute — RCA #14, full trace in the paired charly PR).
    `saveVmDeployState` gains `pruneStaleVmDottedTwin` (self-healing prune
    of a stale dotted-key twin on every write — RCA #6) and preserves a
    prior entry's `VmState.Ephemeral` across a wholesale vm-create write
    (RCA #7) — both regression-tested against real overlay files.
  - `vmshared/cloud_init_render.go`: `resolveCloudInitSSHUser` gains a
    defensive nil guard (RCA #13) — a check command must never SIGSEGV on
    any input, independent of the entity-resolution fix that removed the
    live nil-spec case.
  - `kit/apk_path.go` (NEW): `ResolveApkPath` — the shared apk-path
    resolution helper, relocating a duplicate that used to live
    independently in charly-core's `resolveCheckApk` and the moved android
    plugin preresolve body (R3).

- **Unit 6b** — credential + bed-session R3 consolidation:
  - `deploykit/secret_provision.go` + `enc_passphrase.go` (NEW): the
    credential-store-coupled orchestration relocated from
    `charly/secrets.go`/`charly/enc.go` — `ProvisionPodmanSecrets`,
    `CollectCandySecretAccepts`, `resolveHookSecretEnv`,
    `generateAndStoreSecret`, `EnvVarNameToPodmanSecretSlug`,
    `resolveEncPassphraseForMountWithResolver`, `retryUnavailable`,
    `encNotStoredError` — each taking an injected
    `CredentialAccess{Resolve, Write}` instead of calling charly-core's
    credential store by name, mirroring the existing Cutover B-1
    `secret_probe.go`/`enc_probe.go` precedent exactly.
  - `deploykit/bed_session.go` (NEW): the bed-session orchestration
    relocated from `charly/check_bed_run.go` —
    `PersistBedDeployOverrides`/`DeployNestedLocalChildren`/
    `WaitForVmSshReady`/`WaitForContainerReady`/the bed-level classifier —
    taking the one genuinely provider-registry-coupled classification
    (`bedExternalInPlace`) as a plain injected `bool` parameter rather than
    new wire surface (no cross-process consumer exists today — deliberate,
    per the "no speculative wire surface" discipline).
