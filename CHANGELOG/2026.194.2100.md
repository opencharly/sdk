# 2026.194.2100 — Collection A sdk leg: the resolved-project envelope grows for the S-CM keystone + check-domain seam death

The S-CM keystone (spec half of the K3-D+ program, absorbed into Collection A). Grows the resolved-project
envelope so the check-config seam can DIE and inspect/list can move to plugin-box, and lands the full-width
`spec.CandyModel` build-model the next unit (#60 validate/K3-D) consumes.

## Added

- **`schema/candymodel.cue`** — the serializable candy BUILD model. Four sub-defs CUE-sourced from their former
  hand homes (SDD, wire-types-are-CUE-sourced): `#PackageSection` / `#TagPkgConfig` (were `deploykit.layer_model.go`
  hand structs) + `#EnvConfig` (was `kit.env.go`) + `#RouteConfig` — and `#CandyModel` composing them with
  `#Step`/`#Op`/`#CandyService`/`#CandyExtract`/`#CandyData`/`#CandyApk`/`#CandyVolume`/`#CandyAlias`/`#Shell`/
  `#EnvDependency`. Full-width: identity + plan + lowered ops + resolved package/env/route/shell sections +
  the validate read-surface (Libvirt/Engine/PortRelayPorts/ServiceFiles/env-secret-mcp deps).
- **`schema/resolvedproject.cue`** grows `#ResolvedProject`: `candy_models` (map→#CandyModel), the build
  vocabulary `distro`/`builder`/`init` (`distro`/`init` PIN the existing hand wire structs
  `ResolvedDistro`/`ResolvedInit` via `@go(type=map[string]*…)`, the candy.cue env/apk pattern — no NEW hand
  types; `#Builder` is a clean ref), `templates` (`#ProjectTemplates` — OPAQUE `map[string]RawBody` per kind;
  the host stays kind-blind, the consuming plugin decodes), `agent_bodies` (map→RawBody), `build_targets`
  (`#BuildTarget{Name,Auto}`), `box_plans` (the host-flattened per-box include-ready acceptance plan). Grows
  `#ResolvedBoxView` with box-AGGREGATES (`ports`/`volumes`(`#ResolvedVolumeMount`)/`aliases`/`engine` — the
  `charly box inspect --format` cross-candy values) and `#CandyView` with per-candy `route`/`volumes`/`aliases`/
  `has_init`/`port_relay` (the `charly box list` subcommand sources).

## Changed

- `deploykit/layer_model.go` + `kit/env.go`: the four hand structs DELETED → `type X = spec.X` aliases (R3,
  one CUE source; the projector fills without conversion). Every existing consumer compiles unchanged.

## Removed

- `schema/seam.cue`: `#CheckConfigReply` / `#CheckConfigRequest` (the check-config seam, now dead — plugin-check
  computes the projection off `#ResolvedProject`) and the reply's dead `readiness_json` field. Added
  `#PodDisposableRequest`/`#PodDisposableReply` (the thin per-host-overlay pod-disposability seam).
