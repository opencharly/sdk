# Initial import — the OpenCharly plugin SDK + contract repo

Extracted from the charly module (github.com/opencharly/charly/charly): the
gRPC plugin contract (proto/), the CUE schema source (schema/, embedded FS),
the generated param + wire types (spec/), the plugin-author helpers (kit/), the
go-plugin serve/executor surface (root package sdk), the shared VM types
(vmshared/), the public schema-concatenation contract (schemaconcat/), and the
dev-time codegen (internal/schemagen/).

Every OpenCharly plugin now imports ONLY this module — never charly core. Go
module tags follow v0.<YYYYDDD>.<HHMM> (semver-legal, CalVer-derived), so an
out-of-tree plugin can go get github.com/opencharly/sdk@<tag> — impossible while
the plugin packages lived inside the charly module (charly's v<YYYY.DDD.HHMM>
tags are not valid Go module versions).
