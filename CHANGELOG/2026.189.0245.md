# 2026.189.0245 — aur builder fails loudly on zero produced packages (R8)

The `aur` builder stage ended with
`find /tmp/aur-build -name '*.pkg.tar.zst' -exec cp {} /tmp/aur-pkgs/ \;`, which
SILENTLY succeeds on zero matches. When a package listed under a candy's `aur:`
block has moved from the AUR into a repo, `yay -S --needed` installs it from the
repo WITHOUT building it, makepkg never runs, zero `.pkg.tar.zst` are produced,
and an EMPTY `/tmp/aur-pkgs` COPY layer was committed + cached — a masked
failure that surfaced only downstream (the pgvector /
`check-cachyos-immich-ml-pod` "could not find or read package" build failure).
Added a guard after the copy: if the non-empty package list produced no
artifact, the stage `exit 1`s with a message pointing at the fix (move the
package to the `package:` list). Verified: `TestAurStage_EmptyOutputGuard`
asserts the rendered stage carries the guard after the copy; the guard shell
logic exits 1 on an empty dir, 0 when an artifact is present.
