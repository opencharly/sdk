# 2026.203.1600 — FLOOR-SLIM-proper Unit-8B sdk-contract leg (arbiter holder-dispatch wire trim)

The sdk half of Unit-8B: `candy/plugin-preempt` now dispatches the 6 movable
arbiter host-seams (running/stop/start/switchMode/ensureCDI/gpuCDI) directly
instead of over `ExecutorService.HostArbiter`, so the wire types those seams
carried are dead weight — this leg trims `sdk/spec/arbiter_wire.go` to match.

## Changed

- **`sdk/spec/arbiter_wire.go`** — deleted the `ArbiterSeamRunning`/`Stop`/
  `Start`/`Switch`/`EnsureCDI`/`GpuCDI` action constants and their
  request/reply types (`ArbiterHolderReq`, `ArbiterBoolReply`,
  `ArbiterErrReply`, `ArbiterSwitchReq`, `ArbiterSwitchReply`,
  `ArbiterGpuCDIReply`) — confirmed unused anywhere in the tree (`git grep`,
  both `charly/` and `candy/`) after the paired superproject cutover moved
  every consumer to `sdk.Executor.InvokeProvider`. Only `ArbiterSeamGather`/
  `ArbiterSeamResources` + `ArbiterGatherReply`/`ArbiterResourcesReply`/
  `HolderDescriptor` remain — the 2 seams that stay genuinely host-coupled
  (both `LoadUnified`-project-config-coupled, K1-blocked). Updated the
  file's own header + section comments to describe the 2-seam end state
  instead of the original 7/8-seam design.
- This file is a HAND-WRITTEN wire type (not CUE-generated — `arbiter_wire.go`
  predates the CUE-gen pipeline for this subsystem), so no `task cue:gen`
  regeneration is needed; the trim is a plain Go edit.

## Why

Continues the FLOOR-SLIM-proper core-minimization program (see the paired
superproject `CHANGELOG/2026.203.1600.md` for the full narrative): Unit-8's
crossroad review identified this item as tractable-but-large enough for its
own cutover (Unit B), landing immediately after Unit A. The wire trim keeps
`sdk/spec` describing exactly the seams that still cross the process
boundary — an untrimmed wire contract would silently claim more host
coupling than actually remains, the same class of drift SDD exists to
prevent.
