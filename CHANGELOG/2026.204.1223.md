# 2026.204.1223 — validation-correctness batch (sdk leg): vm PCI-hostdev concreteness prerequisite + deploy-scope `shell:` overlay retirement

The sdk contract-repo leg of the K5-B validation-correctness batch. Two
independent fixes land together because they share one schema-version bump
and one companion `opencharly/charly` PR.

## `#SchemaVersion` bump

`#SchemaVersion` (`schema/version.cue`) bumped from `2026.202.0105` to
`2026.204.1223`. The load-time gate (`LatestSchemaVersion()`) now requires
every `charly.yml` at the new stamp; `charly migrate` brings an older config
forward.

## Deploy-scope `shell:` overlay field removal (the schema half)

`#Deploy.shell` (`schema/deploy.cue`, typed `[...#DeployShellOverlay]`) is
**deleted**, along with the `#DeployShellOverlay` def itself. The field had
**zero live Go consumer**: its only would-be merge, `MergeDeployShell` (fed
by `shellOverlayToEntry`), never had a production call site anywhere in this
project's history — full `git -S` archaeology on the companion `charly` repo
confirmed this back to genesis, not just since the recent dead-code-removal
batch that finally deleted the unreachable helper. Rather than finish a
never-wired half-feature, the honest fix is retiring the authoring surface
outright.

Consequences in this repo:

- `spec/cue_types_gen.go` / `spec/version_gen.go` regenerated via
  `task cue:gen` (`TestGenReproducible` verified fresh).
- `spec/charly_methods.go`: the dead `(*DeployShellOverlay).ByShell()` method
  removed (its sibling `(*Shell).ByShell()`, still live, is unaffected).
- `vmshared/spec_aliases.go`: the `DeployShellOverlay` alias removed.
- `deploykit/capabilities.go`: the `Shell` `CapabilityLabelMap` doc comment
  updated — the label's `Deploy` section is now permanently empty (a stable
  three-section shape a future, properly-designed feature could still
  populate), since its only writer never existed.

## The vm PCI-hostdev concreteness prerequisite

This schema version also carries forward, unchanged in this repo's own
files, the `#LibvirtHostdev` conditional-required-field shape
(`schema/vm.cue`) that the companion `charly` PR's new
`validateKindValueConcrete` narrowly enforces host-side (concreteness was
never something CUE's own closedness-only `Validate()` could catch for a
required-but-unset conditional field — see that PR for the full RCA). No
sdk-side schema change was needed for this half; it is recorded here because
both fixes share this one schema-version bump.
