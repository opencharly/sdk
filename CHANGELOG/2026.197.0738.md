## WalkSeams + ProjectWalker (primary)

Adds the seam contract that lets a plugin (candy/plugin-loader, in the superproject) serve the
whole-project WALK, mirroring the already-landed `DocParser`/`Threaded` seam for the per-document
PARSE (P6). This is the sdk-side half of cutover #46, which kills the last direct
`charly/` → `sdk/loaderkit` import in the superproject.

- `spec/loader_seam.go`: added `WalkSeams` (the six host callbacks — `Parser`, `Boundary`,
  `Threaded`, `ResolveRef`, `GateDoc`, `RepoIdentity` — none of which need the `loaderkit` package)
  and `ProjectWalker` (`WalkProject(rootDir, rootData, rootIdentity, seams) (LoadedProject, error)`)
  — the shared contract home BOTH a loader plugin and the host reference without either importing
  the other, matching the existing header comment's own explanation of why `DocParser`/`Threaded`
  live here rather than in `loaderkit`.
- `loaderkit/walk.go`: deleted the local `WalkSeams` duplicate; `Walk` now takes `spec.WalkSeams`
  directly — ONE canonical type (R3), zero duplication.
