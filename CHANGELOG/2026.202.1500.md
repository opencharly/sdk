## What this leg is

The sdk-contract half of FLOOR-SLIM Unit 4 (CHECK-ENVELOPE + FAMILY): the
check-engine's per-step verdict envelope, formerly a hand-written
`kit.CheckResult` in its entirety, is now split per the operator's approved
embed design (ruling R-FS2):

- **New `sdk/schema/checkresult.cue` → `spec.CheckResult`** (CUE-sourced,
  `task cue:gen`): `Op`/`Verb`/`Status`/`Message`/`Elapsed`/`Attempts`/
  `TotalElapsed`/`CapturedValue` — every field EXCEPT the one already-proven
  (P12 spike) inexpressible exception, `DeadlineExceeded bool json:"-"`.
- **`spec/status_result.go`** (new, hand-written): `Status` (the pass/fail/skip
  int enum + iota consts + `String()`) — gengotypes has no construct for an
  iota enum + Stringer method, so CUE owns the wire value set (a plain `int`,
  referenced via `@go(Status,type=Status)`) and Go owns the formatting
  behavior, mirroring the `#SubstrateKind` split in `status.cue`.
- **`sdk/kit.CheckResult`** now `struct { spec.CheckResult; DeadlineExceeded
  bool `json:"-"` }` — an embedding wrapper, not a hand-duplicated type.
  `kit.Status`/`StatusPass`/`StatusFail`/`StatusSkip` are now type/const
  ALIASES onto `spec.Status`/`spec.StatusPass`/… (kept, deliberately, so
  `kit.Result` and every out-of-process plugin candy's `kit.Pass`/`Fail`/
  `Skip`/`StatusPass` call sites compile unchanged — ~15 external consumers,
  out of scope for this internal floor refactor).
- **`kit.VerbResolver`** (`RunVerb`/`RunProvisionAct`) now returns
  `spec.CheckResult` (not the richer `kit.CheckResult`) — the host-side
  implementer (charly's `hostVerbResolver`) never sets `DeadlineExceeded`
  (an engine-internal retry signal `kit`'s own dispatch loop stamps AFTER the
  verb call returns), so the interface boundary needs only the wire-portable
  base. This is what lets charly core's registry-coupled floor files
  (`provider.go`/`provider_verb.go`/`verb_builtins.go`/`unified_targets.go`/
  `provider_checkenv.go`) reference `spec.CheckResult` directly — zero new
  `sdk/kit` import on the floor.

## Breaking wire-format change — `--format json` / TAP consumers of `charly check box/live/run`

The former hand-written `CheckResult` carried NO json tag on `Op`/`Verb`/
`Status`/`Message`/`Elapsed` (encoding/json defaulted to the bare, capitalized
Go field name); `CheckResult`'s new CUE-sourced base uses deliberate
snake_case keys, matching every other CUE-sourced wire type's convention:

| Old key | New key |
|---|---|
| `Op` | `op` |
| `Verb` | `verb` |
| `Status` | `status` |
| `Message` | `message` |
| `Elapsed` | `elapsed` |

`attempts` / `total_elapsed` / `captured_value` are UNCHANGED (already
snake_case with `,omitempty` before and after). `DeadlineExceeded` never
crossed the wire before and still does not. A new golden test
(`sdk/kit/checkresult_wire_test.go`,
`TestCheckResult_WireParity_Populated`/`_Zero`) proves the diff against the
former shape is EXACTLY this key rename — same field set, same values, same
omission behavior (no unintended omitempty regression on the always-present
fields).
