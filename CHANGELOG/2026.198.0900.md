# 2026.198.0900 — W3 build-engine cone: localpkg + intermediates → sdk/deploykit,buildkit (placeholder, evaluator finalizes)

The sdk producer leg of the W3 build-engine core-minimization wave (consumer:
opencharly/charly `feat/w3-build-engine-plugin`). Two independent host-coupled
subsystems relocated out of charly core after RDD-spike proof that their claimed
host dependency was stale — both operate purely on already-available envelope
data (`CandyModel`/`buildkit.ResolvedBox`), not `*Config`/the live loader graph.

## localpkg subsystem (relocated from `charly/localpkg.go` + partially from
`charly/deploy_host_helpers.go`)

- `deploykit/localpkg.go` (new): `ResolveLocalPkgDir`, `BuildLocalPkgOnHost`,
  `BuildDepPkgsOnHost`, `TransferAndInstallPkgs`, `VenueHasPkgManager`,
  `ExecLocalPkgInstall`, `RenderLocalPkgImageInstall`, `RenderBuilderScript` +
  `hostBuilderContext`. `BuildDepPkgsOnHost`'s ONE genuine core dependency
  (resolving/auto-building a builder IMAGE, which may recurse into `charly box
  build`) is INJECTED as two closures (`resolveImage`/`ensureImage`) rather than
  threaded via `*Config` — the caller (still core) supplies them.
- `deploykit/localpkg_test.go` (new): the pure test suite migrated from
  `charly/localpkg_test.go`, using the existing `localPkgRecExec` DeployExecutor
  fake pattern.
- The `LocalPkg` render-seam (`HostBuild("render-seam", "local-pkg")`) is GONE —
  traced that the caller (`candy_steps.go`) already builds the fully-populated
  step from `dg.Candies`/`dg.Boxes` (the same envelope data) before invoking this
  seam, so the host-side rebuild-from-live-graph was pure redundancy.
  `render_generator_from_project.go`/`render_seam.go` updated; the render-seam
  header comments corrected (3 of 4 original claims — EmitPluginOp,
  EnsureBuilders, InlineBuilder — independently re-verified TRUE: registry/
  plugin-loading is a permanent kernel M-mechanism; only LocalPkg's claim was
  stale).

## intermediates subsystem (relocated from `charly/intermediates.go`)

- `deploykit/intermediates_compute.go` (new): `ComputeIntermediates`,
  `processSiblingGroup`, `walkTrieScoped`, `pickAutoName`, `createIntermediate`,
  `resolvePlatforms`, a new pure `distroBuilderMap` — replacing the `*Config`
  parameter with a new `IntermediateDefaults` struct.
- `buildkit/distro_builder_map.go` (new): `DistroBuilderCandidate` +
  `PickDistroBuilder` — the shared priority-ordered distro-tag-match algorithm,
  extracted so BOTH charly's `Config.distroBuilderMap` (unresolved authored
  config) and deploykit's pure form (already-resolved `boxes` map) adapt their
  own data source onto ONE lookup (R3 — this sharing wasn't possible before the
  move without duplicating the algorithm).
- `deploykit/intermediates_move_test.go` (new): the 1379+155-line test suite
  migrated from `charly/intermediates_test.go` +
  `charly/intermediates_determinism_test.go`, translated onto `CandyModel`/
  `NewSpecCandyModel` fixtures (the established test-fake precedent).
