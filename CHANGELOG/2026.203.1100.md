# 2026.203.1100 — FLOOR-SLIM-proper Unit-8B sdk-contract leg (arbiter holder-dispatch wire trim + CUE conversion)

The sdk half of Unit-8B: `candy/plugin-preempt` now dispatches the 6 movable
arbiter host-seams (running/stop/start/switchMode/ensureCDI/gpuCDI) directly
instead of over `ExecutorService.HostArbiter`, so the wire types those seams
carried are dead weight — this leg trims `sdk/spec/arbiter_wire.go` to match.

**Standing operator directive, applied mid-PR:** any wire type a PR touches
converts to CUE now, regardless of whether it was CUE-sourced before ("a wire
type not generated via CUE before is no excuse to not generate the code from
CUE in freshly committed code"). Since this PR already modifies
`arbiter_wire.go` (a hand-written file predating the CUE-gen pipeline for
this subsystem), the trim above is followed by a full conversion: every
struct in the file is now CUE-sourced, and the hand-written file is deleted
outright.

## Changed

- **`sdk/schema/arbiter.cue`** (NEW) — the CUE source for all 9 arbiter wire
  types: `#HolderAddr`, `#PreemptedHolder`, `#PreemptLease`, `#PreemptLedger`
  (the persisted lease-ledger state), `#HolderDescriptor`,
  `#ArbiterGatherReply`, `#ArbiterResourcesReply` (the 2 remaining
  `HostArbiter` seam replies), and `#ArbiterInvokeInput`/`#ArbiterInvokeReply`
  (the `verb:arbiter` Invoke action-multiplexed wire). None of these are
  open-tailed (no `{known?: string, {[string]: _}}` shape) — every field is
  fully typed and closed, so none qualify for the `hand_state_types.go`
  exception; they convert as plain seam/state structs, per the skill's own
  "plain seam request/reply structs convert, period."
- **`sdk/spec/cue_types_gen.go`** — regenerated via `task cue:gen`, now
  carrying all 9 arbiter types. One generation-driven type change:
  `PreemptLease.OwnerPID` was hand-written `int`; a CUE `int` field
  generates Go `int64` (there is no CUE mechanism to pin a narrower Go int
  width), so it is now `int64`. Every construction/comparison call site
  (`candy/plugin-preempt/arbiter.go`'s two `os.Getpid()` assignments,
  `arbiter_support.go`'s `ownerAlive`/`procStartTime` signatures, and the
  matching test fixtures in `arbiter_test.go`) is updated to match — a
  mechanical width-widening, not a behavior change (a PID always fits in
  either width).
- **`sdk/spec/arbiter_consts.go`** (NEW, hand-written) — the 3 small
  string-enum const groups this domain carries
  (`PreemptStopShutdown`/`PreemptRestoreAlways`/`PreemptRestoreSuccess`,
  `ArbiterSeamGather`/`ArbiterSeamResources`, the 7 `ArbiterAction*` names).
  These are NOT wire-type structs — plain named string literals never carry
  a JSON/YAML shape for `cue exp gengotypes` to generate — so they stay
  hand-written, mirroring the existing `spec/gpu_consts.go` precedent (a
  small hand-written consts file living beside its CUE-generated struct
  siblings).
- **`sdk/spec/arbiter_wire.go`** — DELETED entirely (both the dead-seam trim
  from the first pass of this PR and the surviving 2-seam + persisted-state
  + Invoke types are now generated or moved to `arbiter_consts.go`).
- One deliberate schema-honesty fix during conversion: the original
  `ArbiterInvokeInput.ClaimAddr HolderAddr` field carried `json:"claim_addr,
  omitempty"` — but Go's `encoding/json` `omitempty` is a no-op on a
  non-pointer struct field (a struct value is never considered "empty"), so
  that omitempty never did anything. The CUE def marks `claim_addr!:`
  (required) instead of `?:` — this changes NOTHING about the actual wire
  bytes (the field was always emitted), it only stops the schema from
  claiming an optionality that never existed. `ArbiterInvokeReply.Ledger
  *PreemptLedger` (a genuinely optional pointer) converts straightforwardly
  via `@go(Ledger, optional=nillable)`.

## Why

Continues the FLOOR-SLIM-proper core-minimization program (see the paired
superproject `CHANGELOG/2026.203.1600.md` for the full narrative): Unit-8's
crossroad review identified this item as tractable-but-large enough for its
own cutover (Unit B), landing immediately after Unit A. The wire trim keeps
`sdk/spec` describing exactly the seams that still cross the process
boundary — an untrimmed wire contract would silently claim more host
coupling than actually remains, the same class of drift SDD exists to
prevent. The CUE conversion, triggered by the standing operator directive
mid-review, closes this file's own "still-hand-written *_wire.go is a
conversion-in-progress, not a sanctioned exception" debt at the moment a PR
touches it — the SDD-conversion thematic batch (registered separately)
covers the other 13 untouched hand-written `*_wire.go` files.
