# 2026.194.1727 — K3-A: the build-time BUILDER render engine relocates to sdk/deploykit

2026-07-13

The multi-stage BUILDER render — the detection builders (pixi/npm/aur), the `external_builder:`
out-of-tree builders, the pre-main-FROM stage assembly, and the artifact COPY emitters — relocates OUT
of charly core into `sdk/deploykit`, operating over the `CandyModel` interface + `buildkit.ResolvedBox`.
This is the first K3 (build-engine relocation) unit of the v2 plugin-host program: the render BODIES
leave the kernel; the only host coupling that remains — the provider registry (on-demand connect +
`ResolveBuilder` + `OpResolve` Invoke) — is reached through function-value seam fields on the render
`Generator`, extending the existing `ResolveInlineBuilder`/`EmitPluginOp` seam pattern. The render output
is byte-identical (proven against the box/fedora + box/arch `charly box generate` goldens).

## New

- **`deploykit/builders_render.go`** — the relocated BUILDER render engine over `CandyModel` +
  `buildkit.ResolvedBox`: `EmitPreMainCandyStages`, `EmitBuilderStages`, `EmitExternalBuilderStages`,
  `EmitExternalBuilderArtifacts`, `EmitBuilderArtifacts`, plus the exported `BuilderResolveInputFrom`
  (also consumed by charly's inline-builder seam + the step-emit host builder, R3), the shared
  `DetectExternalizedBuilders` pkg func (the SINGLE detection impl for both the box-build render AND the
  charly deploy build pre-pass, R3), and `expandBuilderPath`.
- **3 registry seam fields on `deploykit.Generator`** (`render_generator.go`), wired host-side by
  charly's `toDeploykit()`: `EnsureBuildersConnected`, `ResolveDetectionBuilderStage`,
  `ResolveExternalBuilderStage` — the render builds the OpResolve input, the host closures do the
  registry `ResolveBuilder` + Invoke and preserve every builder error string byte-exact.
- **`CandyModel.GetExternalBuilder()`** (`candy_model.go`) — the candy's `external_builder:` word (a
  plain, snapshot-safe field read), needed by the relocated `EmitExternalBuilderStages`.

Regeneration on a clean tree is a no-op (no CUE touched; `TestGenReproducible` green). See the
superproject CHANGELOG for the core-side deletion (generate.go −343) + the seam wiring.
