# spec: hostprobe wire types + CredentialHealth home for the externalized charly doctor command

Adds `spec.HostProbeRequest` + `spec.HostProbeReply` (+ `HostProbeDevice`/`HostProbeDistro`) — the
"hostprobe" HostBuild kind envelope — and MOVES `CredentialHealth` here (doctor_wire.go). The
externalized doctor command (candy/plugin-doctor) OWNS the whole host-dependency report and reaches
ONLY the genuine host-hardware subsystem it can't hold (GPU/VFIO/device detection primitives +
credentialHealth + the core install-hint/device tables) via HostBuild("hostprobe"), which returns RAW
FACTS ONLY. CredentialHealth now has core + plugin-doctor consumers (R3, one home; charly core aliases it).
