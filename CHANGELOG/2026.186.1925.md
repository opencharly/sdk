# arbiter wire: add the gpuCDI reverse-channel seam (GPU-reclaim)

Adds `ArbiterSeamGpuCDI` + `ArbiterGpuCDIReply` to `spec/arbiter_wire.go` — the 8th HostArbiter
reverse-channel seam. The host enumerates every RUNNING charly GPU-CDI pod so the arbiter (in
candy/plugin-preempt) can reclaim a leaked/stray one (a kept/failed GPU eval bed whose transient
owner-PID-tied lease died with its check-runner, while the container kept an open nvidia_uvm handle)
BEFORE a nvidia->vfio flip — so the stray can never wedge `modprobe -r nvidia`. Additive wire type;
no schema-version bump (an internal reverse-channel type, not the CUE config schema).
