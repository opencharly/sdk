# K8s substrate-value resolve wire types (Cutover K)

Extend the substrate-value de-type to the `k8s:` cluster template + the k8sgen input:

- `spec/substrate_template_wire.go`: `ResolvedK8s` (`{KubeconfigContext, Raw}` — the
  kernel reads only the context; the full cluster model rides opaquely in Raw),
  `K8sResolveInput/Reply`, and a `K8s` arm on `SubstrateTemplateResolveRequest`.
- `spec/k8sgen_wire.go`: `K8sGenInput.ClusterRaw` (the opaque cluster body) — the
  kernel ships it instead of the typed `Cluster`, and candy/plugin-k8sgen decodes it.

No CUE-schema change (the authored `#K8s` shape is untouched) — no `#SchemaVersion`
bump, no gen-repro drift.
