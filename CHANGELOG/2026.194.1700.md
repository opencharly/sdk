# P14a sdk half — enginekit + the status/oci CUE wire types + OpStatusCollect

Producer half of the P14a superproject cutover (status collectors → command:status;
OCI engine → verb:oci). Landed first (B2 order); the superproject bumps this sdk pointer.

- **sdk/enginekit** (new): `EngineClient` — the host-side podman/docker container-state
  engine (snapshot + inspect) the status collectors consume; `Ports` project to
  `spec.PortMapping`.
- **sdk/schema/status.cue** (new/extended, CUE-sourced — SDD, no hand-written wire structs):
  `#PortMapping`, `#ToolStatus`, `#SubstrateKind` (+ the hand `spec/status_types.go` named
  enum + consts, `@go(-)`), `#DeploymentStatus`; plus the P14a additions —
  `#MergeRequest` / `#MergeReply` / `#ImageUserInput` / `#UserInfo` (schema/oci.cue, the
  verb:oci merge + adopt-user wire), and `#StatusSubstrateRequest` / `#StatusNestedNode` /
  `#StatusSubstrateReply` (schema/status.cue, the HostBuild("status-substrate") seam
  envelope). All generated into `spec/cue_types_gen.go` by `task cue:gen`
  (TestGenReproducible green).
- **sdk/ops.go**: `OpStatusCollect` — the `command:status` programmatic collection op
  (`→ []spec.DeploymentStatus`), distinct from the lifecycle `OpStatus` a substrate plugin
  serves.
