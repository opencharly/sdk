# 2026.197.1320 — check/ADE architecture fold, sdk leg (movable-now sub-batch)

Relocates the genuinely-portable half of the P12-rest RDD trace (charly's
check/ADE surface) into `sdk/kit` and `sdk/deploykit`, per the T-53 trace that
found this batch has a materially larger movable-now surface than the P8b-rest
precedent — no host callback needed for any of the pieces below.

## sdk/kit additions

- `kit/grader.go` (+ `grader_test.go`) — `AgentGrader` (the production
  `StepGrader`), `RunAgentOnce`, `parseVerdict`, `buildGraderPrompt`,
  `lastVerdictObject`, `lastLines`, `GraderDefaultTimeout`. Relocated from
  `charly/check_feature_grader.go`. Lives in `kit`, not `candy/plugin-check`,
  because core CONSTRUCTS an `&AgentGrader{}` value directly
  (`host_build_check_run.go`'s `hostFeatureLive`) — core may only import sdk,
  never a plugin candy, so the concrete type core instantiates must live where
  core can reach it.
- `kit/checkvars.go` (+ `checkvars_test.go`) — `ContainerInspection` +
  siblings, `CheckVarResolver`, `ResolveCheckVarsBuild`,
  `ResolveCheckVarsRuntime`. Every dependency was already portable:
  `spec.BoxMetadata`, `vmshared.BundleNode` (unused param), and a plain
  `{engine} inspect` shell-out.
- `kit/http_do.go` (+ `http_do_test.go`) — `HTTPClientFor`, `DoHTTPRequest`,
  `FormatHTTPHeaders`. Relocated from `charly/check_http.go`, itself
  previously relocated FROM `candy/plugin-http` INTO core solely to share one
  implementation between the in-proc and reverse-channel `HTTPDo` legs — core
  had no core-specific need for this logic at all.
- `kit/planvalidate.go` — `ValidatePlanSteps`. Invoked by BOTH
  `charly box validate` and the externalized `charly feature` HostBuild seam;
  one copy reachable from both without a core→plugin import (R3).
- `kit/local_image.go` (+ `local_image_test.go`) — `LocalImageInfo`,
  `ListLocalImages`, `ParseLocalImagesJSON`, `ResolveLocalImageRef`,
  `ResolveNewestLocalCalVer`, `ExtractCalVerTag`, `LooksLikeFullRef`,
  `ErrImageNotLocal`. `ParseLocalImagesJSON`/`ExtractCalVerTag` are exported
  (charly core's `retention.go` calls them from a different package now);
  `LooksLikeFullRef` had 4 OTHER core callers beyond the moved file — all
  repointed to `kit.LooksLikeFullRef` (R3 single-source).
- `kit/deploy_name.go` — `SanitizeDeployName` (relocated from
  `charly/k3s_post.go`, which had 4 core call sites) + `BareVolumeName`
  (relocated from `sdk/deploykit/quadlet.go` — a pure string helper with no
  deploykit-specific coupling, needed by `checkvars.go`'s
  `mergeRuntimeVars` without a kit→deploykit import, which would cycle).

## sdk/deploykit additions

- `deploykit/check_box_chain.go` — `CheckBoxContainerChain`, the R44
  Option-A box-mode executor. Relocated from `charly/check_box_container.go`
  (imported only `sdk/kit` + stdlib — zero charly-core-only state). Lives
  alongside `ContainerChain` (deploykit already owns that constructor).
  `deploykit/quadlet.go`'s own `BareVolumeName` definition deleted in favor
  of calling `kit.BareVolumeName` (deploykit already imports kit elsewhere in
  this file).

## What did NOT move (per the trace, unchanged)

The GATED-R-ITEM half of the trace (`check_cmd.go`'s live-gather engine,
`check_venue.go`/`check_members.go`'s resolvers, the `host_build_check_bed.go`/
`host_build_check_run.go` session bodies, `description_collect.go`'s
`CollectDescriptions`, etc.) needs the live core `*Config`/`*Generator`/
registry graph and is NOT part of this leg — it joins the tracked K-wave-exit
list (task #21's precedent) for a future wave once the loader/Config state
itself becomes plugin-servable.
