# kit: RemoveImagesByReference — the pod deploy plugin's overlay-image drop (clean externalization)

Adds `kit.RemoveImagesByReference(engineBin, reference)` so the externalized pod deploy plugin
(candy/plugin-deploy-pod) drops its synthesized `<name>-overlay` images in OpPostTeardown itself,
instead of forwarding to a hidden `charly __drop-overlay` core command (deleted in the main repo).

Fixes a PRE-EXISTING destructive bug carried from the old core removeDeployOverlayImages: podman's
`--filter reference=<x>` lists EVERY repo:tag of any matched image ID, so a base image or a cross-deploy
image sharing content leaked into the output — and the blind rmi loop would delete the base + unrelated
deploys' images. `exactRepoRefs` (unit-tested) now re-checks the repository BASENAME in Go and rmi's
ONLY the exact `<reference>` repo. Witnessed: `charly bundle del` of a pod-with-add_candy dropped 18
`check-addcandy-pod-overlay` images (0 remaining) while PRESERVING all 18 base + 21 cross-bed images.
