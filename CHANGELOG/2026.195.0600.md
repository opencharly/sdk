# 2026.195.0600 — F12 live-stdio executor legs (RunInteractive / RunStream)

The sdk-side contract for the F12 interactive/streaming reverse-channel — the live-stdio
siblings of `RunCapture`. Stdio never crosses the plugin wire: the reverse-channel server runs
IN the host `charly` process, so `os.Std*` IS the operator's terminal; only the script + the
exit code travel the wire (the `hostBuildCli` doctrine, generalized off the `charly`-reentry
argv onto the real venue argv).

- **proto** (`proto/plugin.proto`, `ExecutorService`): two new UNARY RPCs
  `RunInteractive(RunRequest) returns (LiveReply)` + `RunStream(RunRequest) returns (LiveReply)`,
  plus `message LiveReply { int32 exit_code = 1; string error = 2; }` — the typed transport
  sibling of `CaptureReply` (a lying reused-buffer reply is avoided). `task proto:gen` clean.
- **CUE carrier** (`schema/seam.cue`): `#PodLiveStdioPlan { script! }` — ONE host-resolved
  live-stdio carrier for shell + cmd + logs (R3 collapse), op-distinguished; `task cue:gen` →
  `spec.PodLiveStdioPlan`, `TestGenReproducible` passes.
- **`spec.DeployExecutor`** (`spec/deploy_executor.go`): `+RunInteractive(ctx, script)(int,error)`
  `+RunStream(...)` + the shared `ErrNotSupported` sentinel (a venue that cannot host a live-stdio
  session fails explicitly, never a silent no-op).
- **`sdk/kit/deploy_executor_live.go`** (NEW): `runLiveExit` (the R3 exit-vs-error mapper — a real
  non-zero exit is the RESULT, not an error) + `ShellExecutor` (`bash -c`, inherited `os.Std*`,
  NO `CommandContext` / NO process-group kill — the TTY owns the lifetime) / `SSHExecutor`
  (`ssh -t` interactive, appending the script only when non-empty so a bare session is the guest
  login shell; `ssh` stream) / `NestedExecutor` (`prepareJump` → delegate to the parent)
  RunInteractive+RunStream.
- **`sdk.Executor`** (`executor.go`): the concrete plugin client `+RunInteractive`/`+RunStream`
  → the proto RPCs.
- **`sdk/ops.go`**: `OpAttach = "attach"` (the F12 interactive/live-stdio Op; `OpShell` re-noted as
  the non-interactive capture leg).
- **regression gate** (`sdk/kit/deploy_executor_live_test.go`): three tests exercising
  `ShellExecutor.RunInteractive/RunStream` inherited-stdout + the `runLiveExit` exit-code contract
  (exit 7/3 returned as the result, not an error). They fail-to-compile without the legs.

Consumers land in the superproject (`charly shell`/`charly cmd` via `OpAttach`; `charly logs
--follow` via `OpLogs` → `RunStream`).
