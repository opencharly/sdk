# 2026.201.0603 — Remove /tmp/cue-rdd scratch-binary residue

The pinned-cue bootstrap and the spec reproducibility test no longer consult
`/tmp/cue-rdd/cue`, a leftover RDD-spike scratch path present since the initial
import. A stale binary at that machine-local path silently won over the
checksum-verified download and the repo's own `bin/cue`, making build and test
inputs host-dependent.

- `Taskfile.yml` `bootstrap-cue` drops the scratch-binary preference branch;
  the checksum-verified pinned download is the only source. Its `desc` no
  longer mentions the scratch binary.
- `spec/gen_repro_test.go` `findCue` drops the `/tmp/cue-rdd/cue` candidate;
  it resolves the repo `bin/cue` or a PATH `cue` reporting the pinned version.
- `schema/README.md` drops the stale claim that the bootstrap prefers a
  pre-fetched scratch binary (the R5 sibling of the removed behavior), and is
  rewritten to the current post-extraction layout: schema travels as
  `schema.FS` consumed via `schemaconcat.ConcatSchema` (no more `//go:embed`
  in charly), generated types live in `spec/` (not `charly/spec`), the
  generator is `internal/schemagen`, the superproject `task cue:gen` wraps
  this repo's own task, and the union-suppression section points at the
  `@go(-)` source instead of an enumerated list that had already drifted.

Verified: `task bootstrap-cue` (edited task) checksum-verifies and installs
`bin/cue` v0.16.1; `GOWORK=off go mod tidy` (no-op) + `go mod verify` +
`go build ./...` + `go vet ./...` + `go test ./...` all pass;
`TestGenReproducible` runs (not skipped) against the repo `bin/cue`.
