The sdk contract leg of the P11c cutover (B2 leg 0). P11c completes the pod deploy
surface's OVERLAY unit (P11 unit c) by DISSOLVING the pod overlay render body OUT of
`charly/` core INTO the compiled-in `candy/plugin-deploy-pod` candy — the candy now
imports `sdk/deploykit` + `sdk/buildkit` + `sdk/kit` DIRECTLY, constructs a
`deploykit.OCITarget` in its own code, and renders the overlay via the shared
`deploykit.NewRenderGeneratorFromProject` helper. The host side shrinks to a
prep + resolve + envelope + cache M-seam (`charly/build_overlay.go`); the candy
side gains the real render (`candy/plugin-deploy-pod/overlay.go`, IMPORT-PURE,
ZERO-ALIASES). This sdk leg carries the THREE new contract surfaces that make the
dissolution possible — the host shrinks because the candy can now reach the render
machinery directly through the sdk.

## The new contract surfaces

- **`deploykit/oci_target.go` (NEW).** The `OCITarget` struct — the kind-blind
  Containerfile walker that emits `# Layer:` headers + delegates each step to its
  `EmitStepOp` — MOVES OUT of `charly/build_target_oci.go` (where it was a core
  struct with per-method bodies) into `sdk/deploykit`, so a plugin candy can
  construct and drive it directly. Alongside it: `OCIEmitStepParams{Dir, StepView,
  PlanView}` — the params an `oci-emit-step` `HostBuild("step-emit", …)` call
  carries (the per-word emitter that the candy's overlay render rides). The walker
  is unit-tested in `deploykit/oci_target_test.go` (emits `# Layer:` + delegates to
  a mock `EmitStepOp`).
- **`deploykit/render_generator_from_project.go` (NEW).**
  `NewRenderGeneratorFromProject(ctx, ex, rp, dir, devLocalPkg)` — the shared render
  helper that builds a render `Generator` from a `spec.ResolvedProject` (envelope
  shape A — reuse the already-resolved project rather than re-resolving host-side).
  The candy's overlay render calls this; the host's remaining render paths call the
  same helper. ONE render entry point, shared, no duplication.
- **`spec/deploy_wire.go` (M — `OverlayBuildReply` extended).** The wire reply the
  host's `HostBuild("overlay")` seam returns to the candy gains the fields the
  candy's render needs to stand up its own `deploykit.OCITarget` WITHOUT a second
  host round-trip: `ResolvedProject`, `Plans`, `BaseUser`, `BaseSecurity`,
  `BaseRegistry`, `CalVer` (the resolved-project + plan + base-image context), the
  `OverlayCandySecurity map[string]*spec.Security` (per-candy security), and
  `ParentVolumes` (the parent-deploy volume slice threaded for overlay nesting).
  This is an EXTENSION of an existing hand-written `*_wire.go` struct (the SDD
  "no new wire type" exception — an existing wire struct gains fields, generated
  by the same `task cue:gen`), NOT a new wire type.
