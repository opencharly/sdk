# 2026.193.1801 — pure-compute relocations from charly core into the sdk libraries (P8b-followup)

The sdk half of #24 (P8b-followup). With the box-build DRIVE now in candy/plugin-build (P8b) and
the RESOLVE/RENDER host-side, the remaining PURE-COMPUTE build helpers relocate from charly core
into the sdk libraries — zero host-coupling (no loader, no `*Config`, no podman, no provider
registry). Behaviour-neutral: `charly box generate` is BYTE-IDENTICAL (verified with a pinned
`--tag` on box/fedora's 46 Containerfiles incl. every auto-intermediate — the intermediate-image
DAG is unchanged).

- **`kit/install_dir.go`** (new) → `kit.InstallDirAtomic(tmp, final string) error` — the pure
  `renameat2(RENAME_EXCHANGE)` atomic dir-swap primitive (was `charly/build_stage_atomic.go`'s
  `installDirAtomic`; `atomicWriteFile` was already `kit.AtomicWriteFile`). `kit` now imports
  `golang.org/x/sys/unix` directly (go.mod: indirect→direct).
- **`buildkit/phase_template.go`** (new) → `buildkit.FormatPhaseTemplate(f *FormatDef, phase, venue)`
  + `buildkit.BuilderPhaseTemplate(b *BuilderDef, phase, venue)` — the pure (phase,venue)→template
  string resolvers (pure over `spec.Format`/`spec.Builder`/`spec.Phase`/`spec.Venue`; the loader
  glue `LoadBuildConfigForBox`/`LoadDefaultBuildConfig` STAYS in charly).
- **`buildkit/system_packages.go`** (new) → `buildkit.RenderSystemPackagesFragment(...)` — the pure
  system-packages install-fragment render (FindFormat → FormatPhaseTemplate → NewInstallContext →
  RenderTemplate). Takes decoded scalars (buildkit cannot import deploykit → no cycle); the
  deploykit-coupled view reconstruction + the `step-emit` host-builder registration STAY in charly.
- **`deploykit/intermediates.go`** (new) → the intermediate-image DAG PURE-COMPUTE funcs, re-typed
  to operate over the `deploykit.CandyModel` interface + `*buildkit.ResolvedBox` (the graph_shim
  pattern): `GlobalCandyOrder`, `PixiBoundCandies`, `GraphReaches`, `TopoSortByPopularity`,
  `SortByPopularity`, `AbsoluteCandySequence`, `RelativeCandySequence`, `ComputeOwnCandies`,
  `AddTransitiveDeps`, `UpdateBoxBase`, `IntersectPlatforms`, `SortedKeys`, `SortedSiblingKeys`,
  `CollectSubtreeBoxes`, `TrieNode`+`NewTrieNode`, `SiblingKey`. The `*Config`-reading orchestration
  (ComputeIntermediates/createIntermediate/processSiblingGroup/walkTrieScoped/resolvePlatforms —
  the RESOLVE half) STAYS in charly and calls these via the core `candyModelMap` adapter.
- **`deploykit/candy_model.go`** — added `HasInstallFiles() bool` to the `CandyModel` interface
  (needed by `PixiBoundCandies`; charly `*Candy` already implements it — sole implementer, a plain
  method on the interface surface, no wire type / no CUE requirement, SDD-clean).

Consumer: the superproject repoints the moved funcs to aliases/shims + deletes the core copies (R5).
