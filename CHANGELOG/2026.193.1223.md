# 2026.193.1223 — build-resolve drive-model wire + the shared image-build lock (P8b sdk half)

The sdk contract half of P8b (the box-build podman DRIVE moves into `candy/plugin-build`; charly's
host side becomes a resolve/render seam provider). See the superproject CHANGELOG for the full move.

- **`sdk/schema/buildresolve.cue`** (new) → generated `spec.BuildResolveRequest` /
  `spec.BuildResolveBox` / `spec.BuildResolveReply` (CUE-sourced per the SDD wire-type mandate). The
  `HostBuild("build-resolve")` seam envelope: the candy's drive marshals a `BuildResolveRequest` (the
  CLI-supplied build inputs + `GenerateOnly`); the host runs the loader RESOLVE + Containerfile RENDER
  + the privileged builder-bootstrap host-side and returns a `BuildResolveReply` drive-model
  (`Engine`/`EngineName`/`Platform`, `Order`⊻`Levels`, per-image `BuildResolveBox` descriptors —
  `FullTag`/`Containerfile`/`Registry`/`Platforms`/`MergeAuto`/`MergeMaxMB`/`MergeMaxTotalMB` — plus
  the resolved `Jobs`/`PodmanJobs`/`Cache`/`KeepImages`). All plain/nested structs — no disjunction,
  no RawBody envelope (`Levels [][]string` generates faithfully via `cue exp gengotypes`).
- **`sdk/kit/filelock.go`** → `ImageBuildLockPath` + `AcquireImageBuildLock`, the pure per-image
  build-lock key derivation + blocking acquire, shared BYTE-IDENTICALLY across the module boundary
  (R3): charly core's `acquireImageBuildLock` and the compiled-in `candy/plugin-build` drive derive
  the same lock path, so a shared intermediate builds cold once while distinct leaves fan out. Mirrors
  the existing `AcquireLocalPkgBuildLock`.

The transitional `spec.MergeRequest`/`spec.MergeReply` the `HostBuild("merge")` seam uses are OWNED by
P14a's `sdk/schema/oci.cue` (PR #28, landing first); this branch rebases onto that and imports them
(no parallel shape — R3).
