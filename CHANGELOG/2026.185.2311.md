# sdk.NewMeta — one shared PluginMetaServer collapsing ~58 hand-rolled metas

Every plugin candy hand-rolled an identical `type meta struct{ pb.UnimplementedPluginMetaServer }`
plus a `Describe` body that just forwarded to `BuildCapabilities(calver, caps, schemaFS, "schema")`
— ~58 byte-identical copies. `sdk.NewMeta(calver, caps, schemaFS) pb.PluginMetaServer` (sdk/meta.go)
collapses them into ONE (R3): a plugin's meta constructor is now a single
`return sdk.NewMeta(calver, caps, schemaFS)` carrying only the genuinely per-plugin data. The schema
dir is fixed to "schema" by convention. Behavior is identical (NewMeta's Describe calls the same
BuildCapabilities). Consumed by the 58-plugin migration in the superproject.
