# sdk — K5-Unit-1: deploy state-model body relocated to sdk/deploykit

## K5-Unit-1 — move the deploy.go BundleConfig state-model BODY into sdk/deploykit (#67 keystone)

The eight deploy state-model functions (`LoadBundleConfig` / `SaveBundleConfig` /
`LoadDeployConfigForRead` / `LoadDeployConfigForWrite` / `MergeDeployOntoMetadata` /
`CleanDeployEntry` / `SaveDeployState` / `ExportAllBox`) plus the `SaveDeployStateInput`
type and the pure helpers (`MarshalBundleNodeLegacy` / `ScopeVolumesToDeployKey` /
`DescriptionInfo` / `IsSameBaseBox` / `RemoveBySource` / `RemoveByExactSource` / `Named`)
relocated from `charly/deploy.go` + `charly/generate.go` + `charly/provides.go` into
`sdk/deploykit` (`deploy_file.go` + `deploy_state.go`).

This is the S-K5 keystone that unblocks P13 (bundle CLI → plugin-bundle cutover):
the deploy state model is now a shared sdk library, callable by both the charly
command family and compiled-in command plugins.

### The seam (the HOW)

Four of the relocated bodies reach core Mechanisms the SDK cannot import — the
unified LOADER (`LoadUnified`), the process-shared deploy-config FLOCK
(`acquireDeployConfigLock`), the legacy→node-form MIGRATE transform
(`migrateDeployEntity`), and the runtime VERSION stamp (`LatestSchemaVersion`).
They reach core through the new `DeployStateHost` seam — a package-level
`*StateHostMechanisms` var in `sdk/deploykit`, filled by `charly/` at `init()`
via `deploykit.RegisterDeployStateHost` (the `DeployConfigPath =
kit.DefaultDeployConfigPath` precedent generalized). Nil-safe: a plugin/SDK
consumer that never writes the per-host ledger leaves the seam nil and the write
paths no-op; the read-only validate/inspect paths work without it.

### ExportAllBox — the #67 keystone

`ExportAllBox` is rewritten to take `*spec.ResolvedProject` (the envelope the
"resolved-project" HostBuild seam produces, K5-Unit-0) instead of the live
`*Config` graph. The box-authored deploy-overlay fields (`description` / `env` /
`env_file` / `security`) now ride `#ResolvedBoxView` (grown in this cutover) —
filled by `projectBoxAggregates` from `cfg.BoxConfig(name)`, alongside the
existing `plan`/`authored_aliases`. The deploy state model is built from the
SAME envelope inspect/list/validate consume — no core `*Config` dep.

### Schema change (SDD)

`sdk/schema/resolvedproject.cue`: `#ResolvedBoxView` grew `description?:` /
`env?:` / `env_file?:` / `security?:` (all existing CUE defs). Regenerated via
`task cue:gen`; `sdk/spec/cue_types_gen.go` carries the new fields
(`Description`/`Env`/`EnvFile`/`Security`). No hand-written wire struct.
