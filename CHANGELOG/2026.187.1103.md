# Derive spec.KindValueDefs from the #<Kind>Value defs (schemagen)

`internal/schemagen` now emits `spec.KindValueDefs` — the word→`#<Kind>Value`
CUE-def map the host uses to closedness-gate a substrate/candy node's authored
value. It is DERIVED from the `#<X>Value` defs themselves (every top-level one
except the shared `#DeployValue` disjunct); `#K8sResourceValues` (plural) does not
match. So the map is single-sourced by the value defs, not hand-maintained.

- New `kindValueDefs(schema)` scans the compiled schema's definitions, matches
  `#<X>Value`, excludes `#DeployValue`, and maps `strings.ToLower(X)` → `#<X>Value`.
- New `writeStrMap` emitter; `KindValueDefs` added to `vocabSets` + `renderVocab`.

Derived output (byte-identical to the former hand map): pod→#PodValue, vm→#VmValue,
k8s→#K8sValue, local→#LocalValue, android→#AndroidValue, candy→#CandyValue.
`TestGenReproducible` green. No `#SchemaVersion` bump (no authored wire-key change).
